# AutoAD-Researcher Step 3.4 开发计划：Idea & Transfer Design

> 文档状态：终审稿 v1.3（v1.2 major_revision_required，本轮收口所有 contracts）
> 更新日期：2026-06-17
> 阶段名称：**Idea & Transfer Design / 科研 Idea 确认与迁移方案设计**
> 上游输入：`stable research_context.json` + `idea_transfer_handoff.json`
> 下游阶段：Step 3.5 Multi-variant Experiment Planner
> 3.4 本身：**不执行代码、不运行实验、不生成文件级 PatchPlan、不重复读取论文/仓库源码**

---

# 0. 阶段定义

```
从 stable 研究上下文出发
  → 用户确认一个科研 Idea（每个 run 恰好一个）
  → 生成 1–3 个架构级 Variant
  → 对每个 Variant 分别判断迁移兼容性
  → 确定性过滤 non-viable / needs_reanalysis Variant
  → 用户选择 Variant + 显式接受风险
  → 确定性验证
  → 交给 3.5 Experiment Planner
```

## 0.1 核心不变量

```
一个 run → 一个 confirmed Idea
一个 Idea → 1–3 个 implementation Variant
Variant 可以选一个或多个
Idea 不得多选
non_viable → 当前 run 终止，新 Idea 使用 child run
```

## 0.2 职责边界

3.4 做：
- 路由 Idea 来源 / 从论文候选展示
- 将用户确认的一个 Idea 转为 `IdeaContract`
- 消费 3.1/3.2 产出；缺失时发 `RepositoryReanalysisRequest` 或 `PaperReanalysisRequest`
- 生成架构级 Variant（使用结构化 tensor delta）
- 逐 Variant 执行 CompatibilityAnalyzer
- 确定性过滤 + 风险聚合
- 用户选择 + 显式接受 risk
- 确定性 Validator（八类不变量）

3.4 不做：
- 重新读论文 PDF（3.2）、重新扫仓库源码（3.1）
- 文件级 PatchPlan（3.6）、开放式 Idea 生成（超出 MVP）
- 执行代码、运行实验、替用户确认任何决策

### Variant.risk_level 聚合

```python
RISK_ORDER = {"low": 1, "medium": 2, "high": 3}

def compute_variant_risk(
    dimension_judgments: list[DimensionJudgment],  # 来自 CompatibilityAnalyzer
    hook_bindings: list[HookBinding],
    regime_changes: list[RegimeChange],
    new_dependencies: list[str],
    hooks: dict[str, ModificationHook],
) -> Literal["low", "medium", "high"]:
    risks: list[str] = []

    for j in dimension_judgments:
        if j.risk:
            risks.append(j.risk)

    for b in hook_bindings:
        hook = hooks.get(b.hook_id)
        if hook and hook.path_classification == "protected_candidate":
            risks.append("high")
        elif hook and hook.path_classification == "unknown":
            risks.append("medium")

    if regime_changes:
        risks.append("medium")

    if new_dependencies:
        risks.append("medium")

    if not risks:
        return "low"

    max_severity_value = max(RISK_ORDER[r] for r in risks)
    for k, v in RISK_ORDER.items():
        if v == max_severity_value:
            return k
    return "low"
```

Validator 必须重算 `risk_level`，与 Variant 中的值比较。不匹配则拒绝。

## 8.2 生成规则

```
1. 同一 Idea 下 1–3 个 Variant
2. hook_bindings 引用 ModificationHook.hook_id
3. 差异维度（至少满足一条）：
   a. 不同 primary_hook
   b. 不同 adapter 策略
   c. 不同 regime_changes
   d. 不同 interface_deltas
4. 不自行声明是否触碰 protected path
5. 不包含文件级信息
```

## 9.3 确定性派生

```python
class ConstraintStrength(str, Enum):
    HARD = "hard"
    USER_CONFIRMED = "user_confirmed"
    SOFT = "soft"

class TransferConstraint(BaseModel):
    constraint_id: str
    category: CompatibilityDimension
    description: str
    strength: ConstraintStrength
    prohibited_changes: list[str]
    required_properties: list[str]
    evidence_ids: list[str]
    user_decision_evidence_id: str | None

class TransferStatus(str, Enum):
    VIABLE = "viable"
    VIABLE_WITH_CONDITIONS = "viable_with_conditions"
    NON_VIABLE = "non_viable"
    NEEDS_REANALYSIS = "needs_reanalysis"


def violates_confirmed_constraint(
    judgment: DimensionJudgment,
    constraints: list[TransferConstraint],
) -> bool:
    """只基于结构化字段判断，不读取 reasoning 自由文本。"""
    for c in constraints:
        if c.strength not in (ConstraintStrength.HARD, ConstraintStrength.USER_CONFIRMED):
            continue
        if c.category != judgment.dimension:
            continue
        for req in c.prohibited_changes:
            if req in judgment.required_changes:
                return True
    return False


def derive_variant_status(
    judgments: list[DimensionJudgment],
    constraints: list[TransferConstraint],
) -> TransferStatus:
    # 按 DIMENSION_POLICY 检查 insufficient_evidence
    for j in judgments:
        if j.status == CompatibilityStatus.INSUFFICIENT_EVIDENCE:
            strategy = DIMENSION_POLICY.get(j.dimension, EvidenceStrategy.SOFT_WARNING)
            if strategy == EvidenceStrategy.REANALYSIS:
                return TransferStatus.NEEDS_REANALYSIS
            elif strategy == EvidenceStrategy.DESIGN_BLOCKING:
                return TransferStatus.NON_VIABLE
            # EXPERIMENT_RESOLVABLE / SOFT_WARNING → 继续检查其他维度

    # INCOMPATIBLE → non_viable（Validator 强制 blocking=true）
    if any(j.status == CompatibilityStatus.INCOMPATIBLE for j in judgments):
        return TransferStatus.NON_VIABLE

    # requires_regime_change + violates constraint → non_viable
    if any(
        j.status == CompatibilityStatus.REQUIRES_REGIME_CHANGE
        and violates_confirmed_constraint(j, constraints)
        for j in judgments
    ):
        return TransferStatus.NON_VIABLE

    # 有 adapter / regime change / insufficient_evidence → viable_with_conditions
    if any(
        j.status in {
            CompatibilityStatus.COMPATIBLE_WITH_ADAPTER,
            CompatibilityStatus.REQUIRES_REGIME_CHANGE,
            CompatibilityStatus.INSUFFICIENT_EVIDENCE,
        }
        for j in judgments
    ):
        return TransferStatus.VIABLE_WITH_CONDITIONS

    return TransferStatus.VIABLE
```

# 10. 确定性过滤

```
viable → 展示
viable_with_conditions → 展示（附带条件）
non_viable → 过滤，记录为 rejected:non_viable
needs_reanalysis → 过滤，记录为 rejected:needs_reanalysis

场景 A: 至少一个 viable/conditional Variant
  → 展示可行 Variant，needs_reanalysis 暂不展示
  → 用户可继续或等待补证据

场景 B: 所有 Variant needs_reanalysis
  → 整体阻塞为 all_variants_need_reanalysis
  → 发出 RepositoryReanalysisRequest / PaperReanalysisRequest
  → 不进入 VariantSelector

场景 C: 所有 Variant non_viable
  → 当前 run 终止为 non_viable
  → 可选 SpawnChildRunRequest

reanalysis 返回后:
  → ResumeFingerprint 触发的所有依赖 artifact 失效重算
  → 重新从 ArchitectureAligner 开始
```


# 14. DeterministicValidator

## 14.1 八类不变量

### (1) Idea 不变量
```
一个 run 一个 confirmed Idea
source_evidence 存在（UserInputEvidenceRef 或 PaperEvidenceRef）
confirmation_status=confirmed
paper_evidence_ids 全部在 Evidence Index 中存在（仅 paper_grounded）
```

### (2) Baseline 不变量
```
baseline_contract 有固定 repository_source_id + commit
所有 hook_id / component_id / phase_id 唯一
所有 evidence_ids 在 Evidence Index 中存在
```

### (3) Variant 不变量
```
数量 ∈ [1, 3]；所有 variant.idea_id 一致
每个 HookBinding.hook_id 存在于 contract.modifiable_hooks
不包含 file-level 字段（files_to_modify 等）
至少满足一条差异维度规则
```

### (4) Variant-Hook 不变量
```
HookBinding 引用的 ModificationHook:
  allowed_for_transfer_design=false → variant non_viable
  path_classification=protected_candidate → variant non_viable
  path_classification=unknown → variant needs_reanalysis
  module_path 在 modifiable_paths.json 中存在
（不依赖 Variant 自报，全部由 hook contract 确定性计算）
```

### (5) Compatibility 不变量
```
每个 variant 有完整 9 维 DimensionJudgment
INCOMPATIBLE → blocking=true（否则拒绝）
DIMENSION_POLICY 维度 insufficient_evidence → 按策略路由
overall_status 严格由 derive_variant_status() 派生
non_viable variant 不进入 VariantSelector
```

### (6) Risk 不变量
```
compute_variant_risk() 重算值必须匹配 Variant.risk_level（否则拒绝）
selected Variant 的 AcceptedRisk 中：
  severity=medium/high → user_decision_evidence_id 非空
  accepted_by_user=true
selected Variant 不能有 unresolved medium/high RiskRecord
```

### (7) VariantSelection 不变量
```
selected 中每个 variant_id 在 viable/conditional 集合中
selected 中每个 SelectedVariant 有 user_decision_evidence_id
非 pending 时 confirmation_status=confirmed
selected 中的 Variant 不包含 non_viable 或 needs_reanalysis
```

### (8) Handoff 不变量
```
source_context_id/version/SHA 匹配上游
idea_contract_sha256 / variant_selection_sha256 匹配
不含 design_blocking unresolved dimension
不含 non_viable 或 needs_reanalysis Variant
next_stage=3.5_multi_variant_experiment_planner
```

```
  → compatibility_analyzed
  → variants_filtered

    → all_variants_need_reanalysis

partial_variants_ready
  → waiting_for_variant_selection

all_variants_need_reanalysis
  → reanalysis_requested

all_non_viable
  → run_terminated_non_viable

  → risks_pending
```

## 23.3 Compatibility 逐 Variant

```
两个 Variant 不同 training status
INCOMPATIBLE + blocking=false → Validator 拒绝
INCOMPATIBLE + blocking=true → non_viable
DATA/LABEL/EVALUATION insufficient_evidence → DESIGN_BLOCKING（non_viable）
RESOURCE insufficient_evidence → EXPERIMENT_RESOLVABLE
SEMANTIC incompatible → DESIGN_BLOCKING
requires_regime_change + violates hard constraint → non_viable
```

## 23.4 Variant-Hook 不变量

```
allowed_for_transfer_design=false → non_viable
path_classification=protected_candidate → non_viable
path_classification=unknown → needs_reanalysis
hook_id 不存在于 contract → 拒绝
module_path 不在 modifiable_paths.json → 拒绝
```

## 23.5 Risk 聚合

```
维度 high risk → 聚合 high
hook protected_candidate → 聚合 high
regime_change → 聚合 medium
new_dependencies → 聚合 medium
compute_variant_risk() ≠ Variant.risk_level → Validator 拒绝
selected variant 缺 AcceptedRisk → 拒绝
AcceptedRisk 无 user_decision_evidence_id → 拒绝
```

## 23.6 VariantSelection

```
selected 中 variant_id 不在 viable/conditional 集合 → 拒绝
SelectedVariant 缺 user_decision_evidence_id → 拒绝
selected 含 non_viable variant → 拒绝
reject_risk 后 selected 为空 → blocked_no_variant_selected
用户沉默不自动选择 → blocked
```

## 23.7 Alignment 来源中立

```
user-provided Idea 无 paper evidence → Alignment 仍合法
IdeaAspectRef.source_kind=user_provided → evidence_ids 为空 ok
AlignmentEntry 含 INCOMPATIBLE + scope=global_idea → non_viable
AlignmentEntry 含 INCOMPATIBLE + scope=specific_hook → 该 hook 不生成 Variant
```

## 23.8 Validator

```
schema_repairable → artifact_repair → 再验证通过
policy_violation → 拒绝，不进入 LLM repair
validate_transfer 不调用 LLM
```
