# AutoAD Research Assistant 上下文驱动改造计划

> **目标**：把 Research Assistant 从「表单向导」改成「上下文驱动科研助手」。
> **核心问题**：当前助手每轮重复追问 dataset/baseline/metric，不看对话历史已确认的事实，方案输出像固定表单（允许修改/禁止修改/验收标准），不像 Claude Code/Aider 那样读上下文、沉淀事实、自然给方案。
> **参考**：Claude Code（Plan Mode 只读探索）、Aider（架构师先读代码再给方案）、Manus（搜索结果不是证据）、MiMoCode（task tree 分步提交）。
> **关联开发计划**：`/root/autodl-tmp/AI4S/参考/用户意图/开发计划.md`
> **关联开发计划章节**：新增 Section 68，联动 Section 67.5。

## 2. Commit 1：上下文沉淀（confirmed_from_chat）

**Goal**：从 transcript 提取用户已确认事实，注入 ResponseContext，禁止 LLM 重复追问。

**Files**：
- 新建 `src/autoad_researcher/assistant/chat_facts.py`
- 修改 `src/autoad_researcher/assistant/intent_action.py`
- 修改 `src/autoad_researcher/ui/research_chat.py`
- 新增 4 个测试

### 2.1 chat_facts.py — rule-based 提取

```python
def extract_confirmed_from_chat(transcript_tail: list[dict]) -> dict[str, Any]:
    all_text = " ".join(
        str(entry.get("content", ""))
        for entry in transcript_tail if entry.get("role") == "user"
    ).lower()
    facts = {}
    if re.search(r'mvtec\s*(ad)?', all_text): facts["dataset"] = "MVTec AD"
    if re.search(r'(patch\s*)?\bcore\b|pathcore|patch\s*core', all_text): facts["baseline"] = "PatchCore"
    if re.search(r'特征提取|backbone|feature\s*extract', all_text): facts["research_direction"] = "feature_extractor"
    metrics = []
    if re.search(r'auroc|auc-?roc|auc\b', all_text): metrics.append("image_level_auroc")
    if re.search(r'pixel.*auc|pixel.*auroc|定位', all_text): metrics.append("pixel_level_auroc")
    if metrics: facts["metrics"] = metrics
    if re.search(r'越高越好', all_text): facts["metric_direction"] = "higher_is_better"
    m = re.search(r'(一天|24\s*(小时|h|hrs|hours?))', all_text)
    if m: facts["budget"] = {"per_candidate_time_limit": "24h"}
    if re.search(r'(不能|不改?|不许|禁止).*(基础框架|框架|core\s*pipeline)', all_text):
        facts["framework_constraint"] = "preserve_patchcore_core_pipeline"
    return facts
```

## 3. Commit 2：自然方案 + scope 分离

**Goal**：research_task_confirmed 不走固定表单，改为 LLM 自然方案；intent draft 不输出文件级 scope。

**Files**：
- 修改 `src/autoad_researcher/assistant/intent_action.py`
- 新增 2 个测试

### 3.1 方案输出结构

不走固定模板，输出自然语言方案包含：已确认事实、研究目标、候选方向、评估计划、执行边界、下一步。

`research_constraints` 保留功能级约束（preserve/allow/forbid 用自然语言），不填具体文件路径。`patch_scope_status: defer_to_patch_planner_after_repo_inspection`。

### 3.3 Tests

- `test_research_task_confirmed_uses_natural_reply`
- `test_intent_draft_does_not_emit_file_level_scope`

## 4. Commit 3：同步 web_search

**Files**：

检测关键词：`搜索论文|搜索方法|最新方法|找代码|web_search|github 实现`。

### 4.2 同步调用

在 `_handle_chat_input` 中 parse_action 之前增加搜索意图分支，同步调用 web_search provider。结果 marking candidate_source_only。provider 不可用返回 search_unavailable。

## 5. 验收条件

| 话术 | 预期 |
|------|------|
| "我想改进 baseline，提升指标" | 不问 dataset，先问缺什么 |
| "MVTec，baseline 是 PatchCore" | 记录为 confirmed，不再重复问 |
| "我想在特征提取器这里发力" | 记录 direction=feature_extractor |
| "一天" | 记录 budget=24h |
| "那我决定完了，下一步？" | 直接生成自然语言方案 |
| "搜索 MVTec AD 最新方法" | 同步返回候选来源或 search_unavailable |
