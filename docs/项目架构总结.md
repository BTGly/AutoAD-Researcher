# AutoAD-Researcher 项目架构总结

> 2026-07-03 · 1,495 tests · ~200+ 源文件 · 3.0–3.9 全部封板 · 3.10 L1/L2 就绪

---

## 1. 定位

面向**工业视觉异常检测**的 **Human-in-the-loop Dry Lab 闭环系统**。

```
论文 + 仓库 → Idea 确认 → Variant 选择 → 实验规划 → Patch 审批 → 受控执行 → 结果分析 → 报告
                  ↑ 用户确认          ↑ 用户确认    ↑ 用户确认                    ↑ 用户确认下一轮
```

MVP：MVTec AD (1-2 类别) · Baseline PatchCore · 单 GPU smoke test · 5 个人工确认闸门

---

## 2. 双路径设计（CLI + Streamlit）

本项目保留 CLI + Streamlit 双路径是**有意设计，不是重复实现**。

**CLI** 是真实 pipeline 执行入口，负责 Stage 3 编排、GPU benchmark、CI 验证和自动化测试。真实 L3 执行必须由终端显式设置 `AUTOAD_L3_REAL_EXECUTION_ALLOWED=1`，避免 UI 误触发高成本或高风险执行。

**Streamlit UI** 是本地 human-in-the-loop 工作台，负责研究意图澄清、artifact 浏览、`input_task.yaml` 生成、approval JSON 写入和 gate 状态展示。UI **不执行** patch-plan、patch-apply、runner-execute 或 stage3-acceptance。

Streamlit 右上角默认菜单中的部署/云服务入口来自框架默认行为，不是项目代码加入的广告。项目通过 `.streamlit/config.toml`（`gatherUsageStats=false`、`toolbarMode=minimal`）和 `st.set_page_config(menu_items=...)` 收敛默认菜单，只保留必要 About 信息。Streamlit 是开源 Python 库，本地运行不需要 Streamlit Cloud 账号或付费服务。

选择 Streamlit 的原因是它适合本地研发控制台：纯 Python、无需前端工程、Session State 内置、artifact/JSON/Markdown/table/chat 组件完善。它不是生产级 Web 应用框架；如果未来要做多用户、远程审批、权限管理和长期运行服务，再考虑 FastAPI + 数据库架构。

## 4. 三层架构

```
┌──────────────────────────────────────────┐
│  AutoAD Core                             │
│  run_id / schema / artifact / 审批 / 白名单 │
│  通过 AgentHarness ABC 调用下层            │
└──────────────────────────────────────────┘
          │ AgentHarness
┌──────────────────────────────────────────┐
│  Harness (2 实现)                         │
│  ├─ SimplePipelineHarness  确定性 no-LLM  │
│  └─ DeepAgentsHarness      真实 Agent    │
└──────────────────────────────────────────┘
          │ 加载 8 Skills
┌──────────────────────────────────────────┐
│  Tool Foundation (8 files)                │
│  ToolSpec → Registry → PermissionEngine   │
│  ProcessTool · Filesystem · Web · GitHub  │
└──────────────────────────────────────────┘
```

- **证据驱动**：UserInputEvidenceRef / PaperEvidenceRef / RepositoryEvidenceRef → Index (JSONL)
- **候选 ≠ 确认**：推荐标记与确认状态分离，沉默 ≠ 授权
- **确定性优先**：Validator 不调 LLM；LLM 输出必经 Validator；retry 最多 2 次且不允许翻案
- **SHA 指纹链**：上游 artifact SHA → 下游验证后消费；Handoff 只含 SHA 引用

## 5. 核心技术

| 选型 | 说明 |
|------|------|
| Python ≥ 3.10 + uv (lockfile) | 精确 pin，拒绝 pip/poetry |
| Pydantic v2, extra="forbid" | 强类型 + field_validator |
| MinerU v0.10.0 | 高精度 PDF 解析，离线 CI fixture |
| subprocess.run(shell=False) | ProcessTool, PermissionEngine 门控 |
| DeepAgents + SimplePipeline | AgentHarness ABC 双实现 |
| Streamlit (本地研发控制台) | 不是生产 Web 框架；纯 Python，零前端，Session State 内置 |
| pytest (离线 fixture) | 不依赖外网 |
| GitHub Actions | verify.sh → pytest → gate |
| LLM: deepseek-v4-flash/pro | fast(tool_use) + primary + fallback(claude-sonnet) |
