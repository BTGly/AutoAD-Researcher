# AutoAD-Researcher Step 3.5 开发计划：Multi-variant Experiment Planner

> 文档状态：初稿 v1.0
> 更新日期：2026-06-17
> 阶段名称：**Multi-variant Experiment Planner / 多变体实验规划器**
> 上游输入：`idea_transfer_design_handoff.json`（来自 3.4 Idea & Transfer Design）
> 下游阶段：Step 3.6 Patch Planner + Step 3.7 Runner / Sandbox
> 3.5 本身：**不生成代码 patch、不执行实验、不修改任何仓库文件**

---

# 0. 阶段定义

Step 3.5 接收 3.4 产出的用户选择的 1–3 个实现 Variant，为其生成**共享评价协议下的统一实验计划包**。

核心不变量：

```
一个 run → 一组共享评价协议（baseline, metric, evaluator, seed, resource）
共享协议 → N 个 Variant 实验计划（N ∈ [1, 3]）
每个 Variant → smoke experiment + full experiment
所有 Variant → 必须按照相同评价口径对比
```

## 0.1 职责边界

3.5 做：
- 从 3.4 handoff 消费已确认 Idea 和用户选择的 Variant
- 从上游 context 提取 baseline 事实（dataset, category, metric, seed）
- 定义共享评估协议（evaluation contract）
- 为每个 Variant 生成结构化实验计划（修改范围、smoke、full、预期、失败条件、资源）
- 生成实验矩阵（variant × stage × seed 的笛卡尔视图）
- 汇总资源预算
- 定义停止策略
- 生成实验验证报告（预执行阶段验证）
- 输出完整的 experiment_bundle.json 和 per-variant plan

3.5 不做：
- 重新读取论文 PDF 或解析方法组件（3.2 已做）
- 重新扫描仓库源码（3.1 已做）
- 重新确认 Idea 或判断兼容性（3.4 已做）
- 生成文件级 PatchPlan（3.6 的职责）
- 执行代码或运行实验（3.7 的职责）
- 替用户静默确认任何决策
- 把多个 Variant 变成完全独立、评价口径不同的实验

# 1. 核心流程

## 1.1 流程总览

```
idea_transfer_design_handoff.json  (← 3.4)
    │
    ├── confirmed_idea          ← Idea 合约
    ├── transfer_analysis       ← 兼容性全貌
    ├── selected_variants       ← 用户选择的 Variant (1–3)
    ├── experiment_resolvable   ← 可在实验中验证的未解决问题
    └── accepted_risks          ← 用户接受的风险
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
  Step 1    Step 2      Step 3
  读取      构建共享    为每个 Variant
  Handoff   评估协议    生成实验计划
    │           │           │
    └───────────┼───────────┘
                ▼
          SharedProtocolDesigner
        (baseline_plan.json 核心)
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
  Variant     Variant     Variant
  Plan A      Plan B      Plan C
                │
                ▼
          ExperimentMatrixBuilder
        (variant × stage × seed)
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
  Resource    Stopping    Validation
  Budget      Policy      Report
                │
                ▼
          ExperimentBundleEmitter
        (输出所有文件)
                │
                ▼
          DeterministicValidator
        (6 类不变量检查)
                │
                ▼
     experiment_planner_handoff.json
        → Step 3.6 Patch Planner
```

## 1.2 流程步骤

```
Step 1: HandoffReader
  读取 idea_transfer_design_handoff.json
  验证上游 artifact SHA 匹配
  提取 confirmed_idea / selected_variants / transfer_analysis / experiment_resolvable

Step 2: SharedProtocolDesigner
  从上游 context + transfer_analysis 提取共享实验条件
  定义 baseline（方法、分数、证据）
  定义共享 metric + evaluator
  定义共享 dataset split
  定义共享 seed / seed range
  输出 baseline_plan.json
  输出 evaluation_contract fingerprint

Step 3: VariantExperimentPlanner（对每个 variant 执行）
  定义 variant 修改范围（从 3.4 ImplementationVariant 消费）
  定义 smoke experiment（目的、最小步骤、成功/失败条件）
  定义 full experiment（完整步骤、超参、预期输出）
  定义失败条件（什么算失败、什么算 noise）
  定义资源消耗预估
  输出 experiment_plans/<variant_id>.json

Step 4: ExperimentMatrixBuilder
  构建 variant × stage × seed 的完整矩阵
  标记共享轴和独立轴
  输出 experiment_matrix.json

Step 5: ResourceBudgetSynthesizer
  汇总所有 variant 的预估资源消耗
  检查是否超出用户预算
  并行/串行调度策略
  输出 resource_budget.json

Step 6: StoppingPolicyDesigner
  定义共享停止条件（超时、连续无提升、NaN 上限）
  定义 per-variant 特有停止条件
  输出 stopping_policy.json

Step 7: ExperimentValidationReporter
  预执行验证：检查计划一致性
  验证共享协议未被任何 variant 破坏
  验证所有 variant 在相同评价口径下可比较
  输出 experiment_validation_report.json

Step 8: ExperimentBundleEmitter
  打包所有输出
  生成 experiment_planner_handoff.json → 给 3.6
```

```python
class HandoffReader:
    """验证上游 artifact 完整性后提取 3.5 所需字段。"""

    def read(self, handoff_path: Path) -> HandoffContext:
        handoff = json.loads(handoff_path.read_text())

        # 1. 验证上游 SHA
        _verify_sha(handoff["source_context_sha256"],
                    self._read_content_sha("research_context.json"))
        _verify_sha(handoff["idea_contract_sha256"],
                    self._read_content_sha("idea_contract.json"))
        _verify_sha(handoff["variant_selection_sha256"],
                    self._read_content_sha("variant_selection.json"))

        # 2. 验证 next_stage 指向正确
        assert handoff["next_stage"] == "3.5_multi_variant_experiment_planner"

        # 3. 提取 3.5 所需字段
        return HandoffContext(
            run_id=handoff["run_id"],
            confirmed_idea=handoff["confirmed_idea"],
            transfer_analysis=handoff["transfer_analysis"],
            selected_variants=handoff["selected_variants"],
            experiment_resolvable=handoff["experiment_resolvable_dimensions"],
            nonblocking_warnings=handoff["nonblocking_warnings"],
            accepted_risks=handoff["accepted_risks"],
        )
```

# 3. Step 2：SharedProtocolDesigner — 共享评估协议

这是 3.5 最核心的设计步骤。它确保所有 Variant 在同一评价协议下可比较。

## 3.1 设计原则

```
1. 所有共享字段必须可追溯到用户提供或用户确认记录
2. 不从内部 Benchmark 配置继承用户任务参数（除非用户明确确认）
3. baseline 和 variant 仅在批准变量上不同
4. evaluation contract 有不可变 fingerprint
```

## 3.2 共享协议内容

```python
class SharedExperimentProtocol(BaseModel):
    """所有 Variant 共享的实验协议。"""

    # --- Baseline ---
    baseline_method: str                          # e.g. "PatchCore"
    baseline_metric_value: float | None           # 如果已测量
    baseline_commit: str                          # 3.1 产出的 commit
    baseline_evidence_ids: list[str]              # RepositoryEvidenceRef

    # --- Dataset ---
    dataset: str                                  # e.g. "MVTec AD"
    categories: list[str]                         # e.g. ["bottle"]
    dataset_split: DatasetSplit                   # train/test 比例/方式

    # --- Metric ---
    primary_metric: str                           # e.g. "image_auroc"
    metric_direction: Literal["higher", "lower"]  # 越高越好还是越低越好
    additional_metrics: list[str]                 # e.g. ["pixel_auroc", "f1"]
    noise_floor: float | None                     # 噪声地板（如果已知）

    # --- Evaluator ---
    evaluator_path: str                           # 评价脚本/模块路径
    evaluator_sha256: str                         # 评价代码的 fingerprint
    evaluator_output_format: str                  # 评价器输出的结构化格式描述
    evaluator_is_protected: bool                  # 必须为 true

    # --- Seed ---
    base_seed: int                                # 共享基础种子
    num_seeds: int                                # 每个实验的种子数（min 1, recommended 3+）

    # --- Resource Budget ---
    max_total_gpu_hours: float
    max_per_experiment_hours: float
    available_gpu_count: int
    available_gpu_type: str                       # e.g. "H100"

    # --- Stopping ---
    stopping_policy_id: str                       # 引用 stopping_policy.json

    # --- Evidence ---
    protocol_evidence_ids: list[str]              # 每个共享值的来源证据
    protocol_fingerprint: str                     # SHA256 of canonical JSON
```

## 3.3 Baseline 值来源解析

```python
class BaselineValueResolver:
    """解析 baseline 的每个值来自哪里。

    优先级：
    1. 用户在 3.2/3.3 中已确认的值 → user_confirmed
    2. 3.1 repository intelligence 检测到的值 → repo_detected（需用户确认）
    3. 论文中提到的值 → paper_mentioned（需用户确认）
    4. 不允许 internal_benchmark_default 静默替代
    """

    def resolve(self, run_id: str) -> dict[str, DecisionSource]:
        context = _read_research_context(run_id)
        baseline_contract = _read_baseline_contract(run_id)

        sources = {}
        for field in ["dataset", "categories", "primary_metric", "base_seed"]:
            sources[field] = _trace_decision_source(context, field)

        # 特殊：baseline_method 来自 3.1 contract
        sources["baseline_method"] = DecisionSource(
            value=baseline_contract.model_name,
            source="repo_detected",
            evidence_ids=baseline_contract.evidence_ids,
        )
        return sources
```

## 3.4 Dataset Split 设计

```python
class DatasetSplit(BaseModel):
    """数据集划分。所有 Variant 共享相同划分以保证可比性。"""

    split_type: Literal["fixed_file", "random_seed", "category_specific"]
    # fixed_file: 使用固定的 train/test 文件（如 MVTec AD 官方划分）
    # random_seed: 使用 base_seed 做随机划分
    # category_specific: 每个 category 独立定义

    train_count: int | None = None
    test_count: int | None = None
    train_ratio: float | None = None             # 如 0.5
    split_file_path: str | None = None            # 如果 split_type=fixed_file
    split_evidence_ids: list[str] = Field(default_factory=list)
```

## 3.5 Evaluation Contract Fingerprint

```python
class EvaluationContract(BaseModel):
    """不可变的评价协议指纹。"""

    contract_id: str
    primary_metric: str
    metric_direction: Literal["higher", "lower"]
    evaluator_path: str
    evaluator_sha256: str
    dataset: str
    categories: list[str]
    dataset_split_sha256: str
    base_seed: int
    num_seeds: int
    protected_paths: list[str]

    # 指纹：上述所有字段的稳定 JSON 序列化的 SHA256
    contract_fingerprint: str

    @staticmethod
    def compute_fingerprint(contract_dict: dict) -> str:
        canonical = json.dumps(contract_dict, sort_keys=True, ensure_ascii=True)
        return hashlib.sha256(canonical.encode()).hexdigest()
```

# 4. Step 3：VariantExperimentPlanner — 每 Variant 实验计划

对 3.4 产出的每个 `ImplementationVariant` 生成独立的实验计划。

## 4.1 设计原则

```
1. 每个 Variant 的实验计划必须引用共享 evaluation_contract
2. 每个 Variant 的 smoke 和 full 实验必须在同一条评价协议下
3. Variant 之间共享 baseline 但独立定义修改范围和失败条件
4. 避免把多个 Variant 变成完全独立、评价口径不同的实验
```

## 4.2 Variant Experiment Plan 结构

```python
class VariantExperimentPlan(BaseModel):
    """单个 Variant 的完整实验计划。"""

    variant_id: str
    variant_label: str
    idea_id: str

    # --- 修改范围（从 3.4 ImplementationVariant 消费）---
    target_hook: str                              # 插入的 hook 位置
    input_interface_change: InterfaceChangeDescription | None
    output_interface_change: InterfaceChangeDescription | None
    adapter_required: bool
    adapter_description: str | None
    regime_change: RegimeChange | None
    state_change: StateChangeDescription | None
    new_dependencies: list[str]
    touches_protected_path: bool                  # 必须为 false

    # --- 共享评估协议引用 ---
    evaluation_contract_ref: str                  # contract_id
    evaluation_contract_fingerprint: str

    # --- 修改约束 ---
    must_preserve_behaviors: list[str]            # 不可改变的行为
    protected_paths: list[str]                    # 不可修改的路径

    # --- Smoke Experiment ---
    smoke: SmokeExperiment

    # --- Full Experiment ---
    full: FullExperiment

    # --- 失败条件 ---
    failure_conditions: list[FailureCondition]

    # --- 资源消耗 ---
    resource_consumption: ResourceConsumption

    # --- 预期输出 ---
    expected_output: ExpectedOutput

    # --- 证据追溯 ---
    evidence_ids: list[str]

class SmokeExperiment(BaseModel):
    """冒烟实验：最小化验证 variant 是否可运行且不破坏 baseline。"""

    purpose: str                                  # e.g. "验证新增 attention 模块不改变 baseline 输出"
    min_steps: list[str]                          # 最小验证步骤
    duration_estimate_minutes: float
    gpu_required: bool
    success_criteria: list[str]                   # e.g. "output shape 不变", "inference time <= 2x"
    failure_criteria: list[str]                   # e.g. "crash", "NaN output", "output shape changed"
    seed_freeze: int                              # smoke 实验固定用此种子
    output_manifest: list[str]                    # 必须产出的文件

class FullExperiment(BaseModel):
    """完整实验：在共享评估协议下跑全量 seed。"""

    purpose: str
    experiment_type: Literal["controlled", "sweep", "comparative"]
    hyperparameters: dict[str, object]            # variant 特有的超参数
    hyperparameters_source: Literal["paper_default", "tuned_for_baseline", "user_specified"]
    steps: list[ExperimentStep]
    duration_estimate_minutes: float
    gpu_required: bool
    expected_training_dynamics: TrainingDynamics | None  # 预期训练动态

class ExperimentStep(BaseModel):
    name: str
    command: str                                  # 逻辑命令（不是 shell 命令，3.7 转换）
    timeout_seconds: int
    produces: list[str]
    depends_on: list[str]                         # 依赖的前置 step name

class FailureCondition(BaseModel):
    condition_id: str
    description: str
    severity: Literal["blocking", "warning", "noise"]
    metric: str | None                            # 哪个指标
    threshold: float | None                       # 阈值条件
    direction: Literal["above", "below"] | None
    action: Literal["stop_variant", "stop_all", "log_and_continue", "flag_for_review"]

class ResourceConsumption(BaseModel):
    estimated_gpu_hours_per_seed: float
    estimated_total_gpu_hours: float              # per_seed * num_seeds
    peak_gpu_memory_gb: float | None
    estimated_cpu_hours: float
    disk_gb: float

class ExpectedOutput(BaseModel):
    primary_metric_expected_range: tuple[float, float] | None
    qualitative_expectation: str                  # e.g. "AUROC 应比 baseline 高 0.5-2%"
    risks: list[str]

class TrainingDynamics(BaseModel):
    expected_loss_curve: Literal["smooth_decay", "plateau_possible", "unstable"]
    expected_convergence_step: int | None
    risk_of_overfit: Literal["low", "medium", "high"]
```

## 4.3 多 Variant 一致性约束

```python
class CrossVariantValidator:
    """确保所有 Variant 在相同评价口径下可比较。"""

    def validate(self, plans: list[VariantExperimentPlan],
                 protocol: SharedExperimentProtocol) -> list[CrossVariantIssue]:
        issues = []

        # 1. 所有 Variant 必须共享同一 evaluation contract
        fingerprints = {p.evaluation_contract_fingerprint for p in plans}
        if len(fingerprints) > 1:
            issues.append(CrossVariantIssue(
                severity="blocking",
                message=f"Variants have divergent evaluation contracts: {fingerprints}"
            ))

        # 2. 所有 Variant 的 primary_metric 必须一致
        metrics = {p.full.experiment_metrics.primary for p in plans}  # conceptual
        if len(metrics) > 1:
            issues.append(CrossVariantIssue(
                severity="blocking",
                message=f"Variants target different primary metrics: {metrics}"
            ))

        # 3. 所有 Variant 的 dataset/categories 必须一致
        # （已由共享协议保证，此处做二次校验）

        # 4. 任何 Variant 的 touches_protected_path 为 true → blocking
        for p in plans:
            if p.touches_protected_path:
                issues.append(CrossVariantIssue(
                    severity="blocking",
                    variant_id=p.variant_id,
                    message="Variant touches protected path"
                ))

        # 5. 确认 Variant 之间有意义的差异
        if len(plans) >= 2:
            hooks = {p.target_hook for p in plans}
            if len(hooks) < len(plans):
                issues.append(CrossVariantIssue(
                    severity="warning",
                    message="Multiple variants target same hook — ensure they differ on adapter/regime/state"
                ))

        return issues

class CrossVariantIssue(BaseModel):
    severity: Literal["blocking", "warning"]
    variant_id: str | None
    message: str
```

# 5. Step 4：ExperimentMatrixBuilder — 实验矩阵

## 5.1 矩阵结构

```python
class ExperimentMatrix(BaseModel):
    """variant × stage × seed 的完整实验矩阵。

    矩阵轴：
      - variant: 用户选择的 Variant（1–3 个）
      - stage: smoke → full（对每个 variant）
      - seed: 1..num_seeds（仅 full 阶段）
    """

    matrix_id: str
    evaluation_contract_fingerprint: str

    variants: list[str]                           # variant_ids
    stages: list[ExperimentStage]
    seeds: list[int]

    # 展开的笛卡尔积表
    entries: list[MatrixEntry]

    # 依赖关系：某些 entry 依赖其他 entry 完成
    dependencies: list[MatrixDependency]

class ExperimentStage(BaseModel):
    stage_name: str                               # "smoke" | "full"
    applies_to_seeds: bool                        # full=true, smoke=false
    is_blocking: bool                             # 失败是否阻止后续

class MatrixEntry(BaseModel):
    entry_id: str                                 # "var_A_smoke_s42"
    variant_id: str
    stage: str
    seed: int | None                              # smoke 无 seed
    shared_axes: list[str]                        # 与其他 entry 共享的轴
    independent_axes: list[str]                   # 此 entry 独有的轴
    depends_on: list[str]                         # 依赖的 entry_ids
    estimated_gpu_hours: float
    priority: Literal["high", "medium", "low"]

class MatrixDependency(BaseModel):
    """矩阵内的执行依赖。"""
    entry_id: str
    depends_on_entry_id: str
    dependency_type: Literal["hard", "soft"]
    # hard: 前置失败 → 此 entry 跳过
    # soft: 前置失败 → 标记 warning 但可继续
```

## 5.2 矩阵展开规则

```python
class MatrixBuilder:
    """从 variant + stage + seed 展开完整实验矩阵。"""

    def build(self, variants: list[VariantExperimentPlan],
              protocol: SharedExperimentProtocol) -> ExperimentMatrix:
        entries = []
        dependencies = []

        for variant in variants:
            # 1. Smoke entry（每个 variant 一个，固定种子）
            smoke_entry = MatrixEntry(
                entry_id=f"{variant.variant_id}_smoke",
                variant_id=variant.variant_id,
                stage="smoke",
                seed=None,
                shared_axes=["evaluation_contract", "dataset", "baseline"],
                independent_axes=["variant_modification", "smoke_check"],
                depends_on=[],
                estimated_gpu_hours=variant.smoke.duration_estimate_minutes / 60,
                priority="high",
            )
            entries.append(smoke_entry)

            # 2. Full entries（每个 variant × num_seeds）
            for seed_idx in range(protocol.num_seeds):
                seed = protocol.base_seed + seed_idx
                full_entry = MatrixEntry(
                    entry_id=f"{variant.variant_id}_full_s{seed}",
                    variant_id=variant.variant_id,
                    stage="full",
                    seed=seed,
                    shared_axes=["evaluation_contract", "dataset", "baseline"],
                    independent_axes=["variant_modification", f"seed_{seed}"],
                    depends_on=[f"{variant.variant_id}_smoke"],
                    estimated_gpu_hours=variant.resource_consumption.estimated_gpu_hours_per_seed,
                    priority="medium",
                )
                entries.append(full_entry)
                dependencies.append(MatrixDependency(
                    entry_id=full_entry.entry_id,
                    depends_on_entry_id=smoke_entry.entry_id,
                    dependency_type="hard",  # smoke 失败 → full 不跑
                ))

        # 3. Baseline entry（如果 baseline 还未测量）
        if protocol.baseline_metric_value is None:
            baseline_entry = MatrixEntry(
                entry_id="baseline_run",
                variant_id=None,
                stage="baseline",
                seed=protocol.base_seed,
                shared_axes=["evaluation_contract", "dataset"],
                independent_axes=[],
                depends_on=[],
                estimated_gpu_hours=0.5,
                priority="high",
            )
            entries.insert(0, baseline_entry)

        return ExperimentMatrix(
            matrix_id=f"matrix_{protocol.run_id}",
            evaluation_contract_fingerprint=protocol.evaluation_contract.contract_fingerprint,
            variants=[v.variant_id for v in variants],
            stages=[
                ExperimentStage("baseline", applies_to_seeds=False, is_blocking=False),
                ExperimentStage("smoke", applies_to_seeds=False, is_blocking=True),
                ExperimentStage("full", applies_to_seeds=True, is_blocking=False),
            ],
            seeds=[protocol.base_seed + i for i in range(protocol.num_seeds)],
            entries=entries,
            dependencies=dependencies,
        )
```

```python
class ExperimentValidator:
    """预执行阶段验证实验包的正确性。"""

    def validate(self, bundle: ExperimentBundle) -> ValidationReport:
        issues = []

        # --- 1. 结构完整性 ---
        issues += self._validate_structure(bundle)

        # --- 2. 共享协议一致性 ---
        issues += self._validate_shared_protocol(bundle)

        # --- 3. 跨 Variant 可比性 ---
        issues += self._validate_cross_variant(bundle)

        # --- 4. 资源约束 ---
        issues += self._validate_resource(bundle)

        # --- 5. 证据追溯 ---
        issues += self._validate_evidence(bundle)

        # --- 6. 停止策略完整性 ---
        issues += self._validate_stopping_policy(bundle)

        return ValidationReport(
            passed=not any(i.severity == "blocking" for i in issues),
            issues=issues,
        )
```

| 类别 | 验证内容 |

| 资源约束 | 总预估 GPU 小时在预算内、单实验在限制内、调度策略可行 |

# 13. 风险与控制

## 风险 1：Variant 评估口径分裂

多个 Variant 如果使用不同的 dataset split、不同的 seed range 或不同的 evaluator，就无法公平比较。

控制：
- `CrossVariantValidator` 强制检查共享协议一致性
- `evaluation_contract_fingerprint` 作为全局不变量
- 验证报告中必须确认所有 variant 共享同一 fingerprint

## 风险 2：Smoke 实验跳过导致 full 实验在错误配置上运行

控制：
- Experiment matrix 中 smoke → full 是 hard dependency
- stopping_policy 中 smoke 失败 → stop_variant

## 风险 3：资源预估不准确导致超预算

控制：
- 每个 Variant 的 `ResourceConsumption` 必须填写
- `ResourceBudgetSynthesizer` 检测超预算并提前告警
- 如果超预算，不阻止 handoff 但标记 `over_budget = true`

## 风险 4：3.4 handoff 的 experiment_resolvable 维度被忽略

控制：
- 3.5 的 `experiment_validation_report.json` 必须列出从 3.4 继承的 `unresolved_from_3_4`
- 验证报告中确认哪些维度将在实验中解决、哪些仍未解决
