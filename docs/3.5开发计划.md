# AutoAD-Researcher Step 3.5 开发计划：Multi-variant Experiment Planner

> 文档状态：修订稿 v1.1（v1.0 major_revision_required，本轮按审阅意见重写）
> 更新日期：2026-06-17
> 阶段名称：**Multi-variant Experiment Planner / 多变体实验规划器**
> 上游输入：`idea_transfer_design_handoff.json`（来自 3.4 Idea & Transfer Design）
> 上游接口状态：`subject_to_3_4_finalization`（P1，通过薄适配层隔离）
> 下游阶段：Step 3.6 Patch Planner + Step 3.7 Runner / Sandbox
> 3.5 本身：**不生成代码 patch、不执行实验、不猜测具体命令或文件名**

---

# 0. 阶段定义

Step 3.5 接收 3.4 产出的用户选择的 1–3 个实现 Variant，为其生成**共享评价协议下的统一实验计划包**。

核心不变量：

```
一个 run → 一个 SharedExperimentProtocol（baseline/meta/evaluator/seed/reuse policy）
共享协议 → N 个 VariantTrialSpec（N ∈ [1, 3]）
每个 Variant → smoke intent + full intent
所有 Variant + baseline → 使用完全相同的 seed 列表
运行前固定 StatisticalAnalysisPlan → 禁止运行后修改成功标准
```

## 0.1 职责边界

3.5 做：
- 从 3.4 handoff 消费已确认 Idea 和用户选择的 Variant
- 设计共享实验协议（引用 3.0 ExperimentInputRefs + 补充评价链覆盖）
- **定义 StatisticalAnalysisPlan**（运行前固定"什么算赢"）
- 为每个 Variant 生成结构化试验规格（smoke/full intent、解析问题、实现需求）
- 生成实验矩阵（baseline × seeds + variant × smoke + variant × seeds）
- 汇总资源预算 + 做预算决策（超预算必须在 3.5 内闭环）
- 定义运行保护策略（operational guard，不含科研提前终止）
- 确定性预执行验证
- 输出实验包 manifest + handoff

3.5 不做：
- 重新读取论文 PDF（3.2 已做）
- 重新扫描仓库源码（3.1 已做）
- 重新确认 Idea 或判断兼容性（3.4 已做）
- 生成文件级 PatchPlan（3.6 的职责）
- 生成具体 shell 命令或猜测脚本文件名（3.6+3.7 的职责）
- 执行代码或运行实验（3.7 的职责）
- 替用户静默确认任何决策
- 把多个 Variant 变成完全独立、评价口径不同的实验
- 运行期间根据已观测指标提前终止 seed（optional stopping）

# 1. 核心流程

## 1.1 流程总览

```
Stage34InputAdapter (薄适配层，隔离 3.4 接口变化)
    │
    ├── confirmed_idea
    ├── selected_variants (1–3)
    ├── transfer_analysis
    ├── transfer_constraints
    ├── experiment_resolvable_dimensions
    └── accepted_risks
                │
                ▼
        SharedExperimentProtocol
          - 引用 3.0 ExperimentInputRefs
          - 补充评价链覆盖检查
          - baseline reuse policy
          - 显式 seed 列表
                │
                ▼
        StatisticalAnalysisPlan
          - aggregation / dispersion / paired-by-seed
          - minimum meaningful effect
          - missing run policy
          - multiple variant comparison
          - decision rules
                │
                ▼
        VariantTrialSpecs
          - per-variant smoke intent
          - per-variant full intent
          - implementation requirements
          - hyperparameter plan（防泄漏）
                │
                ▼
        ExperimentalResolutionPlans
          - 每个 3.4 unresolved dimension → 可执行验证步骤
                │
                ▼
        ExperimentMatrix
          - baseline × seeds
          - variant × smoke（固定 seed）
          - variant × seeds
          - 依赖 DAG
                │
                ▼
        ResourceBudget + BudgetDecision
          - 资源估算（含估算来源和置信度）

          - 不含科研提前终止
                │
                ▼
        Deterministic Plan Validator
          - 8 类不变量检查
                │
                ▼
        ExperimentPlannerHandoff
          - 引用 9 个 artifacts 的 SHA
          - 不嵌套完整对象
                │
                ▼
        Step 3.6 Patch Planner
```

## 1.2 上游适配层

```python
class Stage34InputAdapter(Protocol):
    """薄适配层，隔离 3.4 handoff 格式变化。

    3.4 handoff 字段最终封板后，只需修改此 adapter 的实现，
    Planner 核心逻辑不变。
    """

    def load_confirmed_idea(self) -> IdeaContract: ...
    def load_selected_variants(self) -> list[ImplementationVariant]: ...
    def load_transfer_analysis(self) -> TransferAnalysis: ...
    def load_transfer_constraints(self) -> list[TransferConstraint]: ...
    def load_resolvable_dimensions(self) -> list[UnresolvedDimension]: ...
    def load_accepted_risks(self) -> list[AcceptedRisk]: ...
```

## 1.3 风险说明

3.4 的 `transfer_analysis` 是 Idea 级别公共分析（在 VariantGenerator 之前）。3.5 可以消费公共 analysis，但 Variant 独有的 `regime_change`、`state_change`、`adapter_required` 等约束必须从 `selected_variants` 中分别编译到 per-variant trial specs，不能假设所有 Variant 的兼容性完全相同。

# 2. Step 1：SharedExperimentProtocol — 共享实验协议

## 2.1 设计原则

```
1. 复用 3.0 已封板的 ExperimentInputRefs，不新建平行合同体系
2. 补充评价链中未被现有指纹间接覆盖的组件
3. baseline 和所有 Variant 使用完全相同的 seed 列表
4. 旧 baseline 结果只有在所有输入 fingerprint 完全匹配时才能复用
```

## 2.2 Schema

```python
class SharedExperimentProtocol(BaseModel):
    """所有 Variant 共享的实验协议。"""

    model_config = ConfigDict(extra="forbid")

    protocol_id: str
    schema_version: Literal[1]

    # --- 引用 3.0 基础设施 ---
    experiment_input_refs: ExperimentInputRefs
    # 包含: repository_fingerprint, environment_sha256,
    #       dataset_manifest_sha256, asset_manifest_sha256, command_sha256

    # --- 评价链补充引用（仅在被现有指纹间接覆盖时才可为 None）---
    supplemental_refs: SupplementalEvaluationRefs

    # --- Baseline ---
    baseline_method: str
    baseline_config_sha256: str
    baseline_policy: BaselineExecutionPolicy

    # --- 共享实验参数 ---
    seeds: list[int]                              # 显式列表，不用 base_seed+i 推导
    primary_metric: str
    metric_direction: Literal["maximize", "minimize"]

    # --- 共享约束 ---
    protected_paths: list[str]                    # 来自 3.4 transfer_constraints
    must_not_change: list[InterfaceConstraint]

    # --- 证据 ---
    protocol_evidence_ids: list[str]
    protocol_fingerprint: str

class SupplementalEvaluationRefs(BaseModel):
    """补充引用：仅在未被子项 ExperimentInputRefs 间接覆盖时必填。

    覆盖规则：
      - evaluator 位于固定 repository commit 内 → repository_fingerprint 间接覆盖
      - metric parser 位于固定 repository commit 内 → repository_fingerprint 间接覆盖
      - post-processing 由仓库代码+固定配置+command plan 决定 → repository_fingerprint + command_sha256 覆盖
      - dataset split 已包含在 dataset_manifest_sha256 中 → 不增加独立 ref
    """
    evaluator_ref: ArtifactRef | None = None
    metric_parser_ref: ArtifactRef | None = None
    postprocessing_ref: ArtifactRef | None = None
    dataset_split_ref: ArtifactRef | None = None

class InterfaceConstraint(BaseModel):
    """不可改变的接口约束。"""
    reason: str
    contract_description: str

class BaselineExecutionPolicy(BaseModel):
    """Baseline 执行与复用策略。"""

    mode: Literal["reuse_existing", "run_fresh"]

    # --- 复用条件 ---
    reuse_source: BaselineResultRef | None = None
    # 仅在 mode=reuse_existing 时提供

    # --- 运行策略 ---
    seeds: list[int]                              # baseline 使用的 seed 列表
    per_seed_timeout_seconds: int | None = None

class BaselineResultRef(BaseModel):
    """引用已存在的 baseline 结果。所有身份字段必须完全匹配才能复用。"""

    source_run_id: str
    repository_fingerprint: str
    baseline_config_sha256: str
    dataset_manifest_sha256: str
    evaluation_contract_sha256: str
    environment_lock_sha256: str
    asset_manifest_sha256: str
    command_sha256: str
    seeds: list[int]
    per_seed_metrics: list[SeedMetric]

class SeedMetric(BaseModel):
    seed: int
    metric_name: str
    metric_value: float
```

## 2.3 Evaluator 身份补充说明

`SupplementalEvaluationRefs` 的验证逻辑：

```python
def validate_evaluation_chain_coverage(
    protocol: SharedExperimentProtocol,
    repo_path: Path,
) -> list[ValidationIssue]:
    """验证评价链每个组件是否被现有指纹直接或间接覆盖。"""
    issues = []
    refs = protocol.supplemental_refs

    # evaluator: 检查是否在 repository 内
    evaluator_in_repo = _evaluator_in_repository(
        protocol.protected_paths, repo_path
    )
    if not evaluator_in_repo and refs.evaluator_ref is None:
        issues.append(ValidationIssue(
            severity="blocking",
            message="Evaluator not covered by repository_fingerprint and no supplemental ref"
        ))

    # metric parser: 同上
    # post-processing: repository_fingerprint + command_sha256 是否覆盖
    # dataset split: dataset_manifest_sha256 是否覆盖

    return issues
```

# 3. Step 2：StatisticalAnalysisPlan — 统计分析计划

## 3.1 设计原则

```
1. 运行前固定所有判断规则，禁止运行后修改成功标准
2. baseline 和 Variant 使用相同 seed，允许配对比较
3. 明确缺失 seed 或失败 run 的处理策略
4. MVP 不要求复杂显著性检验，但至少应：
   - 报告逐 seed 指标、均值和离散程度
   - 预先规定最小有意义改进量
   - 禁止运行后临时改变成功标准
```

## 3.2 Schema

```python
class StatisticalAnalysisPlan(BaseModel):
    """运行前固定的统计分析计划。

    这个文件单独计算 SHA，防止运行后修改判断标准。
    """

    model_config = ConfigDict(extra="forbid")

    plan_id: str
    schema_version: Literal[1]
    protocol_fingerprint: str

    # --- 核心指标 ---
    primary_metric: str
    metric_direction: Literal["maximize", "minimize"]

    # --- 聚合方式 ---
    aggregation: Literal["mean", "median"]
    dispersion: Literal["std", "iqr"]

    # --- 比较方式 ---
    paired_by_seed: bool = True
    # True: 逐 seed 配对比较 (baseline_seed_i vs variant_seed_i)
    # False: 聚合后比较

    # --- 最小有意义改进 ---
    minimum_meaningful_effect: float | None = None
    # None = 纯描述性，不做"显著"判定
    # 非 None = delta 低于此值视为 practical equivalence
    minimum_meaningful_effect_source: str | None = None
    # e.g. "paper_reported_baseline_std", "domain_expert_estimate", "user_specified"

    # --- 缺失数据处理 ---
    missing_run_policy: Literal[
        "invalidate_variant",
        "rerun_failed_seed_once",
        "report_incomplete",
    ]
    max_rerun_attempts: int = 1

    # --- 多 Variant 比较 ---
    multiple_variant_policy: Literal[
        "descriptive_only",
        "rank_by_primary_metric",
    ]
    # MVP 建议 descriptive_only

    # --- 判定规则 ---
    decision_rules: list[ScientificDecisionRule]

    plan_fingerprint: str  # SHA256 of canonical JSON

class ScientificDecisionRule(BaseModel):
    """一条科学判定规则。"""

    rule_id: str
    description: str
    condition: DecisionCondition
    conclusion: str
    # e.g. "Variant A is likely beneficial", "No meaningful difference detected"

class DecisionCondition(BaseModel):
    condition_type: Literal[
        "all_seeds_improved",
        "mean_improved_above_threshold",
        "paired_improved_majority",
        "descriptive_only",
    ]
    parameters: dict[str, object] = Field(default_factory=dict)
```

# 4. Step 3：VariantTrialSpecs — Variant 试验规格

## 4.1 设计原则

```
1. 不生成具体 shell 命令、不猜测脚本文件名
2. 只声明实验意图（action intent）+ 输入/输出需求
3. Hyperparameter 规划必须防止测试集泄漏
4. 真实命令由 3.6 Patch Planner + 3.7 Runner 基于仓库入口生成
```

## 4.2 Schema

```python
class VariantTrialSpecs(BaseModel):
    """所有 Variant 的试验规格（单文件）。Variant 仅有 1–3 个，单文件便于原子校验。"""

    model_config = ConfigDict(extra="forbid")

    specs_id: str
    schema_version: Literal[1]
    protocol_fingerprint: str

    variants: list[VariantTrialSpec]

class VariantTrialSpec(BaseModel):
    """单个 Variant 的完整试验规格。"""

    variant_id: str
    variant_label: str
    idea_id: str

    # --- 从 3.4 ImplementationVariant 消费 ---
    target_hook: str
    regime_change: RegimeChange | None
    state_change: StateChangeDescription | None
    adapter_required: bool
    new_dependencies: list[str]

    # --- 受保护约束（来自 3.4 transfer_constraints）---
    protected_paths: list[str]
    # 注意：3.5 不自行判断是否触碰——
    # 3.5 只传递约束，真正路径验证在 3.6 PatchPlan Validator 中完成

    # --- Smoke 试验意图 ---
    smoke: TrialIntent

    # --- Full 试验意图 ---
    full: TrialIntent

    # --- 实现需求（交给后续 environment/asset builder）---
    implementation_requirements: VariantImplementationRequirements

    # --- Hyperparameter 规划 ---
    hyperparameter_plan: HyperparameterPlan

    # --- 证据 ---
    evidence_ids: list[str]

class TrialIntent(BaseModel):
    """实验意图：声明做什么，不写具体命令。"""

    intent_id: str
    intent_type: Literal[
        "baseline_run",
        "smoke_inference",
        "variant_fit",
        "full_evaluation",
    ]
    description: str

    # 输入需求
    required_inputs: list[ArtifactRequirement]
    # 输出需求
    expected_outputs: list[ArtifactRequirement]

    # 约束
    timeout_policy_ref: str
    depends_on: list[str]                         # 依赖的 intent_id

class ArtifactRequirement(BaseModel):
    """一个输入或输出需求。"""
    artifact_type: Literal[
        "model_weights",
        "config_file",
        "dataset_split",
        "evaluation_results",
        "training_log",
        "metrics_json",
    ]
    description: str
    format_hint: str | None = None                # e.g. "PyTorch .pth", "JSON with keys [...]"

class VariantImplementationRequirements(BaseModel):
    """Variant 对环境、资产和资源的额外需求。3.5 只声明，不构建。"""

    dependency_deltas: list[DependencyRequirement]
    asset_requirements: list[AssetRequirement]
    accelerator_requirements: AcceleratorRequirement
    environment_rebuild_required: bool

class DependencyRequirement(BaseModel):
    package_name: str
    version_spec: str | None = None               # e.g. ">=1.0,<2.0"
    reason: str

class AssetRequirement(BaseModel):
    asset_description: str                        # e.g. "Pretrained ResNet-50 weights"
    expected_source: Literal["torchvision_hub", "huggingface", "user_provided", "url"]
    expected_sha256: str | None = None
    required: bool = True

class AcceleratorRequirement(BaseModel):
    gpu_required: bool
    min_vram_gb: float | None = None
    gpu_type_preference: str | None = None        # e.g. "H100", "L40S"

class HyperparameterPlan(BaseModel):
    """Hyperparameter 规划，防止测试集泄漏。"""

    mode: Literal[
        "fixed_from_source",     # 直接使用论文/仓库中的默认值
        "fixed_by_user",         # 用户显式指定
        "predeclared_search",    # 预先声明的搜索（需 validation split）
    ]
    source_evidence_ids: list[str]

    # --- 固定参数 ---
    parameters: list[HyperparameterSpec] = Field(default_factory=list)

    # --- 搜索空间（仅 mode=predeclared_search）---
    search_space: list[SearchParameter] | None = None
    selection_split: DatasetPartition | None = None
    # 必须明确是哪个 split 用于选择超参数——禁止使用 test split
    search_budget: SearchBudget | None = None
    selection_metric: str | None = None

class HyperparameterSpec(BaseModel):
    name: str
    value: object
    source: Literal["paper_default", "user_specified", "baseline_inherited"]
    rationale: str | None = None

class DatasetPartition(BaseModel):
    partition_name: str                           # e.g. "validation", "train[80%]"
    is_test_split: bool                           # 必须为 False
    evidence_ids: list[str]

class SearchBudget(BaseModel):
    max_trials: int
    max_gpu_hours: float

class SearchParameter(BaseModel):
    name: str
    type: Literal["float", "int", "categorical"]
    range: list[object] | tuple[float, float]
```

## 4.3 Hyperparameter 防泄漏规则

```text
规则 1：禁止在 test split 上选择超参数
  → selection_split.is_test_split 必须为 False
  → 无合法 validation split 时，只允许 mode=fixed_from_source 或 fixed_by_user

规则 2：不同 Variant 的搜索预算必须一致或显式声明差异
  → 如果 Variant A 搜索 20 trials、Variant B 不搜索，必须在报告中标记

规则 3：搜索空间在运行前冻结
  → search_space 写入 trial spec 后不可修改
```

## 4.4 3.4 transfer_analysis 的使用方式

3.4 的 `transfer_analysis` 是 Idea 级别的公共分析。3.5 按以下方式消费：

```text
transfer_analysis.dimensions[]
  ├── status=compatible → 不生成额外约束
  ├── status=requires_regime_change → 写入 VariantTrialSpec.regime_change
  ├── status=compatible_with_adapter → 写入 VariantTrialSpec.adapter_required
  └── status=incompatible → 该 Variant 不应到达 3.5（3.4 的 VariantGenerator 已过滤）

transfer_constraints[]
  ├── protected_paths → 写入 SharedExperimentProtocol 和 VariantTrialSpec
  ├── must_not_change → 写入 SharedExperimentProtocol
  └── hard_constraints → 写入 VariantTrialSpec 的实现需求
```

# 5. Step 4：ExperimentalResolutionPlans — 实验解析计划

## 5.1 设计原则

```
每个 3.4 传入的 experiment_resolvable dimension
→ 恰好一个可执行的 resolution plan
→ 声明由哪个实验 entry 验证、观察什么量、怎样判定通过/拒绝
```

## 5.2 Schema

```python
class ExperimentalResolutionPlans(BaseModel):
    """将 3.4 的未解决问题编译为可执行的验证步骤。"""

    model_config = ConfigDict(extra="forbid")

    plans_id: str
    schema_version: Literal[1]
    protocol_fingerprint: str

    resolutions: list[ExperimentalResolutionPlan]
    # 必须覆盖所有来自 3.4 的 experiment_resolvable_dimensions
    # 数量必须相等，且每个 dimension 恰好一个 resolution

class ExperimentalResolutionPlan(BaseModel):
    unresolved_dimension_id: str                  # 来自 3.4 UnresolvedDimension
    dimension: str                                # CompatibilityDimension 名称
    variant_id: str                               # 由哪个 Variant 验证
    verification_stage: Literal["smoke", "full"]

    # 对应的矩阵 entry
    target_entry_ids: list[str]

    # 观察量
    observable: str                               # e.g. "attention_loss_convergence"
    observation_source: str                       # e.g. "full.steps[0].output.fit_log.json"

    # 判定标准
    acceptance_criterion: Criterion
    rejection_criterion: Criterion

    # 结果如何影响 downstream
    resulting_decision: str
    # e.g. "If attention module converges → training dimension resolved as compatible"

class Criterion(BaseModel):
    criterion_type: Literal[
        "value_in_range",
        "value_above_threshold",
        "value_below_threshold",
        "no_nan_detected",
        "output_shape_matches_baseline",
    ]
    parameters: dict[str, object] = Field(default_factory=dict)
```

## 5.3 Validator 强制约束

```python
def validate_resolution_coverage(
    resolutions: list[ExperimentalResolutionPlan],
    unresolved: list[UnresolvedDimension],
) -> list[ValidationIssue]:
    """每个 unresolved dimension 必须有恰好一个 resolution plan。"""
    issues = []
    resolved_ids = {r.unresolved_dimension_id for r in resolutions}
    unresolved_ids = {u.dimension_id for u in unresolved}

    missing = unresolved_ids - resolved_ids
    extra = resolved_ids - unresolved_ids

    for mid in missing:
        issues.append(ValidationIssue(
            severity="blocking",
            message=f"Unresolved dimension {mid} has no resolution plan"
        ))
    for eid in extra:
        issues.append(ValidationIssue(
            severity="warning",
            message=f"Resolution plan references unknown dimension {eid}"
        ))
    return issues
```

# 6. Step 5：ExperimentMatrix — 实验矩阵

## 6.1 设计原则

```
1. baseline 和所有 Variant 使用完全相同的 seeds 列表
2. smoke 使用固定 seed（通常 base_seed），不展开多 seed
3. 依赖关系只在 depends_on 中表达，不重复
4. 不包含具体 shell 命令
```

## 6.2 Schema

```python
class ExperimentMatrix(BaseModel):
    """baseline × seeds + variant × smoke + variant × seeds 的完整矩阵。"""

    model_config = ConfigDict(extra="forbid")

    matrix_id: str
    schema_version: Literal[1]
    protocol_fingerprint: str

    seeds: list[int]
    variants: list[str]    # variant_ids

    entries: list[MatrixEntry]
    # 依赖关系通过 entry.depends_on 表达，不另设顶层 dependencies 列表

class MatrixEntry(BaseModel):
    entry_id: str                                 # e.g. "baseline_s42", "var_A_smoke", "var_A_full_s42"
    variant_id: str | None                        # None = baseline entry
    stage: Literal["baseline", "smoke", "full"]
    seed: int | None                              # smoke 为固定 seed，但仍记录具体值

    # 实验意图引用
    intent_ref: str                               # 引用 VariantTrialSpec 中的 intent_id

    # 依赖
    depends_on: list[str]                         # 依赖的 entry_ids
    # smoke 依赖 baseline（如果 baseline 尚未有复用结果）
    # full 依赖对应 variant 的 smoke

    # 共享轴 vs 独立轴
    shared_axes: list[str]                        # 与其他 entry 共享的轴
    independent_axes: list[str]                   # 此 entry 独有的轴

    # 资源
    estimated_gpu_hours: float
    priority: int = 0                             # 0 = 最高

class MatrixDependency(BaseModel):
    """矩阵内的执行依赖。"""
    entry_id: str
    depends_on_entry_id: str
    dependency_type: Literal["hard", "soft"]
```

## 6.3 展开规则

```python
class MatrixBuilder:
    """从 shared protocol + variant specs 展开矩阵。"""

    def build(self, protocol: SharedExperimentProtocol,
              specs: VariantTrialSpecs) -> ExperimentMatrix:
        entries = []
        seeds = protocol.seeds

        # 1. Baseline entries（每个 seed 一个）
        baseline_policy = protocol.baseline_policy
        if baseline_policy.mode == "run_fresh":
            for seed in seeds:
                entries.append(MatrixEntry(
                    entry_id=f"baseline_s{seed}",
                    variant_id=None,
                    stage="baseline",
                    seed=seed,
                    intent_ref="baseline_run",
                    depends_on=[],
                    shared_axes=["protocol", "dataset", "metric"],
                    independent_axes=[f"seed_{seed}"],
                    estimated_gpu_hours=0.5,
                    priority=0,
                ))

        # 2. Variant entries
        for variant in specs.variants:
            # 2a. Smoke（固定 seed）
            smoke_seed = seeds[0]
            entries.append(MatrixEntry(
                entry_id=f"{variant.variant_id}_smoke",
                variant_id=variant.variant_id,
                stage="smoke",
                seed=smoke_seed,
                intent_ref=variant.smoke.intent_id,
                depends_on=(
                    [f"baseline_s{smoke_seed}"]
                    if baseline_policy.mode == "run_fresh" else []
                ),
                shared_axes=["protocol", "dataset", "metric", "baseline"],
                independent_axes=["variant_modification", "smoke_check"],
                estimated_gpu_hours=0.1,
                priority=0,
            ))

            # 2b. Full（每个 seed 一个）
            for seed in seeds:
                entries.append(MatrixEntry(
                    entry_id=f"{variant.variant_id}_full_s{seed}",
                    variant_id=variant.variant_id,
                    stage="full",
                    seed=seed,
                    intent_ref=variant.full.intent_id,
                    depends_on=[f"{variant.variant_id}_smoke"],
                    shared_axes=["protocol", "dataset", "metric", "baseline"],
                    independent_axes=["variant_modification", f"seed_{seed}"],
                    estimated_gpu_hours=1.0,
                    priority=1,
                ))

        return ExperimentMatrix(
            matrix_id=f"matrix_{protocol.protocol_id}",
            schema_version=1,
            protocol_fingerprint=protocol.protocol_fingerprint,
            seeds=seeds,
            variants=[v.variant_id for v in specs.variants],
            entries=entries,
        )
```

# 7. Step 6：ResourceBudget + BudgetDecision — 资源预算与决策

## 7.1 设计原则

```
1. 超预算必须在 3.5 内闭环——以下状态禁止 handoff：
   - revision_required
   - override_pending
   - override_rejected
2. 资源估算必须有来源说明（estimate_source），不允许 LLM 猜测无依据数字
3. 预算决策状态机在 3.5 内完成
```

## 7.2 Schema

```python
class ResourceBudget(BaseModel):
    """资源预算汇总 + 预算决策。"""

    model_config = ConfigDict(extra="forbid")

    budget_id: str
    schema_version: Literal[1]
    protocol_fingerprint: str

    # --- 用户预算上限 ---
    limits: ResourceLimits

    # --- Per-variant 汇总 ---
    per_variant: dict[str, VariantResourceSummary]

    # --- 总计 ---
    total_estimate: ExperimentBundleResourceBudget

    # --- 预算决策 ---
    budget_decision: BudgetDecision


class ResourceLimits(BaseModel):
    max_total_gpu_hours: float
    max_per_experiment_gpu_hours: float
    available_gpu_count: int
    available_gpu_type: str

class EntryResourceEstimate(BaseModel):
    """单个矩阵 entry 的资源估算。"""
    entry_id: str
    variant_id: str | None
    stage: str
    seed: int | None
    estimated_gpu_hours: float
    estimated_peak_gpu_memory_gb: float | None
    estimated_disk_gb: float
    estimate_source: str     # e.g. "baseline_historical", "paper_reported", "rough_guess"
    confidence: Literal["high", "medium", "low"]
    assumptions: list[str]

class VariantResourceSummary(BaseModel):
    """单个 Variant 的资源汇总。"""
    variant_id: str | None    # None = baseline
    entries: list[EntryResourceEstimate]
    total_gpu_hours: float
    total_wall_clock_hours: float  # 考虑依赖链的最长路径

class ExperimentBundleResourceBudget(BaseModel):
    total_gpu_hours: float
    total_wall_clock_hours: float
    max_single_experiment_gpu_hours: float

class BudgetDecision(BaseModel):
    """预算决策状态机。"""

    status: Literal[
        "within_budget",
        "revision_required",
        "override_pending",
        "override_confirmed",
        "override_rejected",
    ]

    original_limits: ResourceLimits
    estimated_consumption: ExperimentBundleResourceBudget
    utilization_pct: float

    over_budget_items: list[str] = Field(default_factory=list)

    # --- 降级选项（仅 revision_required 时提供）---
    revision_options: list[BudgetRevisionOption] = Field(default_factory=list)

    # --- 用户决策（仅 override_confirmed/rejected 时有效）---
    user_decision_evidence_id: str | None = None
    decided_at: datetime | None = None

class BudgetRevisionOption(BaseModel):
    option_id: str
    description: str                              # e.g. "Reduce seeds from 5 to 3"
    savings_gpu_hours: float
    new_total_gpu_hours: float
    tradeoffs: list[str]
```

## 7.3 预算决策状态机

```
resource_estimated
        ↓
budget_evaluated
   ├── within_budget
   │     → validation → handoff
   │
   ├── over_budget
   │     → revision_required
   │        ├── 减少 seed 数量
   │        ├── 减少 Variant 数量
   │        ├── smoke-only（跳过 full）
   │        └── 降低资源配置（更小的模型/更少的 epoch）
   │
   └── override_available
         → override_pending
         → 用户显式 confirm
            ├── override_confirmed → handoff
            └── override_rejected → 回到 revision_required
```

**进入 3.6 的必要条件：`status == within_budget` 或 `status == override_confirmed`**

**禁止 handoff 的状态：`revision_required`、`override_pending`、`override_rejected`**

# 8. Step 7：OperationalGuardPolicy — 运行保护策略

## 8.1 设计原则

```
1. 只处理运行安全条件，不做科研决策
2. 不包含根据已观测指标提前终止 seed 的逻辑
3. 科研重复实验必须完成预声明的所有 seed
```

## 8.2 Schema

```python

class PlanValidator:
    def validate(self, run_dir: Path) -> ExperimentPlanValidationReport:
        issues = []

        # 1. 结构完整性
        issues += self._validate_structure(run_dir)

        # 2. Baseline 公平性
        issues += self._validate_baseline_fairness(run_dir)

        # 3. StatisticalAnalysisPlan
        issues += self._validate_stat_plan(run_dir)

        # 4. Hyperparameter 防泄漏
        issues += self._validate_hyperparameter_safety(run_dir)

        # 5. 跨 Variant 一致性
        issues += self._validate_cross_variant(run_dir)

        issues += self._validate_budget_closure(run_dir)

        # 8. 评价链覆盖
        issues += self._validate_evaluation_chain(run_dir)

        blocking = [i for i in issues if i.severity == "blocking"]
        return ExperimentPlanValidationReport(
            report_id=f"validation_{run_dir.name}",
            passed=len(blocking) == 0,
            issues=issues,
        )
```

# 14. 风险与控制

## 风险 1：Baseline 复用条件不满足但仍尝试复用

控制：
- `BaselineExecutionPolicy` 强制检查 9 项身份指纹
- Validator 第 2 类不变量检查

## 风险 2：StatisticalAnalysisPlan 在运行后被修改

控制：
- `plan_fingerprint` 覆盖所有决策参数
- 独立的 `statistical_analysis_plan.json` 单独计算 SHA
- 运行结束后校验 plan SHA 是否与运行前一致

## 风险 3：Hyperparameter 在 test split 上选择

控制：
- `HyperparameterPlan.selection_split.is_test_split` 必须为 False
- Validator 第 4 类不变量检查
- 无合法 validation split → 只允许 fixed 模式

## 风险 4：ExperimentActionIntent 过于抽象导致 3.6 无法转换

控制：
- Intent 包含具体的 `required_inputs` 和 `expected_outputs`
- `ArtifactRequirement` 提供 `format_hint` 辅助下游理解
- 3.6 开发时，根据实际仓库结构反向验证 intent 的可翻译性

## 风险 5：超预算但用户不在场

控制：
- `revision_required` 状态提供自动降级选项（减少 seed、减少 Variant、smoke-only）
- 系统可以推荐降级方案但不自动执行
- `waiting_for_budget_override` 允许持久化等待用户返回

# 19. 最终阶段定义

Step 3.5 完成后可以声明：

> AutoAD-Researcher 能够在接收 3.4 产出后，设计共享实验协议（复用 3.0 infrastructure + 补充评价链覆盖），运行前固定 StatisticalAnalysisPlan（聚合/离散/配对/最小效应/缺失策略），为每个 Variant 生成 intent-only 试验规格（含防泄漏 hyperparameter plan），将 3.4 的未解决问题编译为可执行 resolution plan，构建 baseline×seeds + variant×smoke + variant×seeds 完整矩阵，汇总资源预算并完成预算决策（超预算在 3.5 内闭环），定义运行保护策略（不含科研提前终止），通过 8 类不变量验证，最终产出 SHA-only 引用的 handoff 交给 Patch Planner。

```
已生成代码 patch（那是 3.6 的工作）
已执行实验（那是 3.7 的工作）
已验证迁移效果（那是 3.8 的工作）
已替代用户做 Variant 选择（那是 3.4 的工作）
已从零设计科研 Idea（那是 3.4 的工作）
已生成具体 shell 命令或脚本名（那是 3.6+3.7 的工作）
```
