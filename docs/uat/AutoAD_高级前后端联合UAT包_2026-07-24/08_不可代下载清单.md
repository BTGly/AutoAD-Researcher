## 需要用户自行提供

真实 PatchCore/MVTec 物理流程涉及下列许可或硬件前置条件，不能由测试 Agent 自动下载、重新分发或伪造：

1. **MVTec AD 数据集**：由用户从官方渠道取得；不得提交到 Git、测试包或共享 runs。
2. **固定 PatchCore checkout**：按项目 readiness 文档规定的仓库和 commit，使用干净、独立目录。
3. **离线 backbone 权重**：使用 readiness 指定文件和 SHA-256。
4. **可用 NVIDIA GPU**：物理验收前先运行 `nvidia-smi` 和项目 readiness checker。
5. **受许可论文 PDF**：付费或机构访问材料由用户依法取得。

若任一物理前置条件缺失，系统应报告 external blocker，不得把 CPU fixture、历史快照或 mock 结果描述成真实物理实验。

## 关于 DOCX、PDF 和仓库 ZIP

原始聊天资料包包含合成 DOCX、PDF 和仓库 ZIP，用于模拟产品上传。GitHub 分支采用 AI 可读、可审查的 Markdown 源材料和 fixture 生成脚本，不提交重复二进制文件。需要真实浏览器上传二进制附件时，可由测试者根据 `generated/01_spike_ad_two_stage/source_materials/` 生成临时 DOCX/PDF/ZIP，并将其放在独立测试目录；不得把临时生成文件作为产品事实或提交到 `main`。
