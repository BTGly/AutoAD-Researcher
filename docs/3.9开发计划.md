# 0. 阶段定义

```
实验结果是否支持迁移 Idea？
实验设计是否可以得出结论？
失败是否可以归因？
报告是否可追溯？
```

```
experiment_execution_handoff.json
    ↓
提取 baseline 指标 vs Variant 指标
    ↓
指标方向检查（提升/下降/持平）
    ↓

    ↓
运行失败归因
    ↓
资源成本比较
    ↓
结果是否支持 Idea？
    ↓
结果是否足以得出结论？
    ↓
生成 Reflection
    ↓
输出 final_report.md
```

# 2. 分析维度

## 2.1 Baseline 与 Variant 对比

```python
class MetricComparisonEntry(BaseModel):
    metric_name: str
    direction: Literal["maximize", "minimize"]
    baseline_value: float
    variant_value: float
    delta: float
    delta_pct: float
    improved: bool  # 基于 direction 判断
    significant: bool  # 基于 absolute_tolerance 判断

class ComparisonReport(BaseModel):
    entries: list[MetricComparisonEntry]
    improved_count: int
    degraded_count: int
    unchanged_count: int
    summary: str  # LLM 生成的对比总结
```

## 2.2 指标方向检查

```
对于每个 metric：
  direction = maximize → variant_value > baseline_value → improved
  direction = minimize → variant_value < baseline_value → improved

如果改进方向不符合预期 → 标记 degraded
如果变化在 tolerance 内 → 标记 unchanged
如果所有指标 unchanged → 实验无区分度 → inconclusive
```

## 2.3 评价协议一致性

```
检查 baseline attempt 的 evaluation contract SHA
  是否等于 Variant attempt 的 evaluation contract SHA？
  不等 → evaluation_protocol_diverged → validity 降级
  等 → evaluation_protocol_consistent
```

## 2.4 运行失败归因

```python
class FailureCategory(str, Enum):
    PREFLIGHT_FAILURE = "preflight_failure"      # 环境/数据问题
    EXECUTION_CRASH = "execution_crash"           # 代码 bug
    TIMEOUT = "timeout"                           # 资源不足
    METRIC_PARSE_FAILURE = "metric_parse_failure" # 输出格式不兼容
    REPRODUCIBILITY_FAILURE = "reproducibility_failure"  # 两次 attempt 不一致
    VALIDITY_FAILURE = "validity_failure"         # 科学有效性不通过

class FailureAnalysis(BaseModel):
    attempt_id: str
    category: FailureCategory
    root_cause_summary: str
    evidence_ids: list[str]
    actionable: bool  # 是否可以修复后重试
    suggested_fix: str | None
```

## 2.5 资源成本比较

```python
class ResourceComparison(BaseModel):
    baseline_runtime_seconds: float | None
    variant_runtime_seconds: float | None
    baseline_peak_memory_mb: float | None
    variant_peak_memory_mb: float | None
    overhead_pct: float | None  # Variant 相对于 baseline 的额外开销
    within_budget: bool
```

## 2.6 结果是否支持 Idea

```
Idea 声称的预期效果：
  从 idea_contract.transfer_relevance 提取

实际结果：
  从 ComparisonReport 提取

判断逻辑（LLM + 规则）：
  - 所有目标指标 improved → strongly_supported
  - 大部分目标指标 improved、个别 unchanged → partially_supported
  - 全部 unchanged → inconclusive（需要更多实验或更大差异）
  - 有 degraded → not_supported
  - 实验失败 → cannot_judge
```

## 2.7 结果是否足以得出结论

```
sufficient：
  - 两次 attempt 都成功
  - reproducibility_report.status = reproducible
  - validity_report.status = valid
  - metrics_report.status = passed

insufficient：
  - 任一 attempt 失败
  - reproducibility 未通过
  - validity 不通过
  - 所有指标 unchanged（效应太小）

partial：
  - 部分指标有改进，部分 unchanged
  - 需要进一步实验才能确定
```

## 3.2 Reflection

```python
class Reflection(BaseModel):
    idea_supported: Literal["strongly_supported", "partially_supported", "not_supported", "inconclusive", "cannot_judge"]
    conclusion_sufficient: bool
    key_learnings: list[str]
    unexpected_results: list[str]
    limitations: list[str]
    next_steps: list[str]
    recommended_actions: list[Literal[
        "iterate_on_current_idea",  # 继续优化
        "try_alternative_variant",   # 换 Variant
        "try_alternative_idea",      # 换 Idea
        "investigate_failure",       # 先排查失败
        "publish_or_share",          # 结果足以分享
    ]]
```

```markdown
# AutoAD-Researcher 实验报告

## 1. 实验概要

## 2. Idea 与迁移设计
- Idea 来源

## 4. 结果对比
- 指标对比表
- 提升/下降分析

## 5. 科学有效性
- Validity 检查结果
- 评价协议一致性
- 数据泄漏检查
- 局限性

## 6. 结论
- Idea 是否得到支持
- 结论是否充分
- 下一步建议

## 7. 附录
- 完整 artifact 清单
- SHA256 指纹
- Evidence Index 引用
```
