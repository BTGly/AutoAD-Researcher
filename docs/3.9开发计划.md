# 0. 阶段定义

```
3.8 ExperimentExecutionHandoff + 3.5 StatisticalAnalysisPlan + SharedExperimentProtocol
    ↓
EvidenceSufficiency (per-variant seed pair 覆盖)
    ↓
PairedMetricObservation 表 (seed-keyed)
    ↓
AggregatedMetricComparison (按 aggregation 聚合)
    ↓
逐 Variant ScientificConclusion (执行 StatisticalAnalysisPlan 规则)
    ↓
Idea-support 确定性映射 (first-match priority)
    ↓
ValidityInterpretation (聚合 3.0 reports → scientific_validity_aggregate.json)
    ↓
FailureHypothesis (含 evidence)
    ↓
ResourceComparison (budget_status: within/over/not_assessable)
    ↓
Reflection + NextRunProposal
    ↓
report_facts.json → final_report.md → ReportFactValidator
```

# 1. 核心模型

## 1.1 PairedMetricObservation（seed-keyed）

```python
class PairedMetricObservation(BaseModel):
    seed: int

    baseline_unit_id: str
    baseline_metric_ref: ArtifactRef
    baseline_value: float

    variant_unit_id: str
    variant_metric_ref: ArtifactRef
    variant_value: float

    delta: float
    validity_refs: list[ArtifactRef]
```

列表顺序不能作为证据合同——必须显式绑定 seed。

## 1.5 Idea-support 决策表（first-match priority）

规则按优先级排序，第一条匹配即返回：

```
1. 存在 beneficial 且 存在 worse
   → implementation_sensitive
   (同一个 Idea 在不同 Variant 中方向相反)

2. 全部 beneficial
   → consistently_supported

3. 至少一个 beneficial，且无 worse
   → supported_by_at_least_one_variant

4. 全部 worse
   → not_supported_by_tested_variants

5. 全部 incomplete / cannot_judge
   → cannot_judge

6. 存在 practically_equivalent 或 mixed，且无 beneficial 无 worse
   → not_demonstrated_or_partial

7. 其他
   → cannot_judge
```

此表是确定性的。LLM 只能撰写解释文本。

## 1.6 ValidityInterpretation

```python
class ValidityInterpretation(BaseModel):

```

输出文件：`scientific_validity_aggregate.json`（不与 3.0 的 `scientific_validity_report.json` 同名）。

## 1.7 ResourceComparison

```python
class ResourceComparison(BaseModel):
    variant_id: str
    dimension: Literal["wall_time", "peak_rss", "peak_gpu_memory", "gpu_seconds"]
    comparison_status: Literal[
        "comparable",
        "missing_baseline", "missing_variant",
        "measurement_unavailable",
        "incompatible_measurement",
    ]
    baseline_value: float | None
    variant_value: float | None
    delta: float | None

class ResourceComparisonReport(BaseModel):
    per_variant: dict[str, list[ResourceComparison]]
    budget_status: Literal["within_budget", "over_budget", "not_assessable"]
    measurement_quality: Literal["complete", "partial", "unavailable"]
```

`budget_status=not_assessable` 当 measurement unavailable 时使用，不改 `bool` 到 `None`。

```python
class FailureHypothesis(BaseModel):
```

## 1.10 报告生成

```python
class ReportFacts(BaseModel):
    run_id: str; idea_label: str; baseline: str; dataset: str
    per_variant_conclusions: dict[str, ScientificConclusion]
    idea_support: str
    comparisons: list[AggregatedMetricComparison]
    validity_aggregate: ValidityInterpretation
    failure_analysis: FailureAnalysis | None
    resource_comparison: ResourceComparisonReport | None
    artifact_registry: list[ArtifactRef]
    evidence_sufficiency: dict[str, EvidenceSufficiency]

class ReportFactValidation(BaseModel):
    passed: bool
    issues: list[str]  # 正向：篡改数值/SHA/conclusion
                        # 反向：遗漏 invalidity/failure
```

# 3. 测试（新增）

```
[ ] beneficial+worse → implementation_sensitive (rule 1)
[ ] beneficial+equivalent → supported_by_at_least_one_variant (rule 3, 无 worse)
[ ] all beneficial → consistently_supported (rule 2)
[ ] all worse → not_supported (rule 4)
[ ] all incomplete → cannot_judge (rule 5)
[ ] seed 42 baseline 与 seed 42 variant 显式配对在 PairedMetricObservation
[ ] measurement_unavailable → budget_status=not_assessable
[ ] measurement_unavailable → 不编造 within_budget=True
[ ] reused baseline 在报告中标记 baseline_source=reused
[ ] final_report 篡改 ScientificConclusion → ReportFactValidator 拒绝
[ ] final_report 遗漏 invalidity → ReportFactValidator 拒绝
[ ] output named scientific_validity_aggregate.json (not scientific_validity_report.json)
```
