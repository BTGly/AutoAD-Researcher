```

    ↓

    ↓

    ↓

    ↓

    ↓

    ↓
report_facts.json（确定性）
    ↓

    ↓

```

# 1. 与 3.5 预注册规则的关系

3.5 已定义：

```python
class StatisticalAnalysisPlan(BaseModel):
    minimum_meaningful_effect: float
    equivalence_margin: float
    aggregation: Literal["mean", "median"]
    dispersion: Literal["std", "iqr"]
    paired_by_seed: bool
    minimum_completed_pairs: int

class ScientificConclusion(str, Enum):
    BENEFICIAL = "beneficial"
    PRACTICALLY_EQUIVALENT = "practically_equivalent"
    MIXED = "mixed"
    WORSE = "worse"
    INCOMPLETE = "incomplete"
```

3.9 不自行定义阈值或结论类型。它**执行** 3.5 的规则：

```
输入：all seed-paired metrics
执行：StatisticalAnalysisPlan.deterministic_decision_conditions
输出：ScientificConclusion（3.5 定义的枚举）
```

`ScientificConclusion` → 科研语义映射是确定性的：

```
BENEFICIAL              → "结果支持 Idea"
PRACTICALLY_EQUIVALENT  → "未证明差异"
MIXED                   → "部分指标提升，部分下降"
WORSE                   → "结果不支持 Idea"
INCOMPLETE              → "数据不足以得出结论"
```

# 2. 核心模型

## 2.1 MetricObservationKey（多 seed/Variant/dataset_row 的 key）

```python
class MetricObservationKey(BaseModel):
    variant_id: str | None  # baseline 时为 None
    metric_name: str
    dataset_row: str
    seed: int
```

## 2.2 PerObservationMetric（单次观测）

```python
class PerObservationMetric(BaseModel):
    key: MetricObservationKey
    unit_id: str
    parsed_metric_ref: ArtifactRef  # 引用 3.8 的 ParsedMetric
    value: float | None             # None = parse failed
    parse_status: Literal["parsed", "missing", "invalid"]
    source_sha256: str
```

## 2.3 AggregatedMetricComparison（聚合对比）

```python
class AggregatedMetricComparison(BaseModel):
    variant_id: str
    metric_name: str
    dataset_row: str

    baseline_values: list[float]  # 来自 baseline×seeds
    variant_values: list[float]   # 来自 variant×seeds
    paired_count: int

    aggregation_method: str             # 来自 StatisticalAnalysisPlan
    baseline_aggregate: float | None
    variant_aggregate: float | None
    dispersion: float | None

    effect_size: float | None           # variant - baseline (for maximize)
    relative_change_pct: float | None   # 100 * effect / baseline

    comparison_status: Literal[
        "comparable",
        "missing_baseline", "missing_variant",
        "invalid_baseline", "invalid_variant",
        "insufficient_pairs",
        "single_seed_only",
    ]
```

`comparison_status ≠ comparable` 的 entry 不参与 ScientificConclusion 判定，但必须出现在报告中。

## 2.4 ScientificConclusionReport（执行 3.5 计划）

```python
class ScientificConclusionReport(BaseModel):
    statistical_analysis_plan_sha256: str
    executed_rules: list[str]  # 实际执行的决策规则 ID

    overall_conclusion: ScientificConclusion  # 3.5 枚举

    per_variant_conclusions: dict[str, ScientificConclusion]
    per_metric_conclusions: dict[str, ScientificConclusion]

    conditions_for_beneficial: list[str]  # 未满足的条件（如果 applicable）
    incomplete_reasons: list[str]         # 如果 INCOMPLETE

    evidence_refs: list[ArtifactRef]
```

结论不来自 LLM，来自确定性地执行 `StatisticalAnalysisPlan.deterministic_decision_conditions`。

## 2.5 ValidityInterpretation（不是替代 3.0 report，是解读）

```python
class ValidityInterpretation(BaseModel):
    per_unit_validity: dict[str, ScientificValidityReport]  # 引用 3.8 已有

    protocol_consistency: Literal["consistent", "diverged"]
    protocol_divergence_details: list[str]

    evaluation_contract_fingerprint: str  # 来自 SharedExperimentProtocol

    overall_validity: Literal[
        "all_valid",
        "partial_invalid",
        "requires_manual_review",
    ]

    evidence_refs: list[ArtifactRef]
```

## 2.6 FailureHypothesis（有 evidence 的归因）

```python
class FailureHypothesis(BaseModel):
    hypothesis_id: str
    unit_id: str
    category: FailureCategory
    claim: str
    status: Literal["confirmed", "inferred", "unknown"]
    confidence: Literal["high", "medium", "low"]
    evidence_refs: list[ArtifactRef]
    falsification_steps: list[str]

    alternative_hypotheses: list[str]
```

## 2.7 FailureAnalysis

```python
class FailureAnalysis(BaseModel):
    failed_unit_ids: list[str]
    hypotheses: list[FailureHypothesis]

    execution_errors: list[str]       # 确定性归因（exit code、timeout、stderr pattern）
    inferred_causes: list[str]        # LLM 辅助归因

    actionable_failures: list[str]    # 可修复
    blocked_failures: list[str]       # 需要上游重新介入
```

## 2.8 ResourceComparison

```python
class ResourceComparison(BaseModel):

    baseline_avg_runtime: float | None
    variant_avg_runtime: dict[str, float | None]

    overhead_pct: dict[str, float | None]

```

## 2.9 Reflection + NextRunProposal（只提议，不执行）

```python
class NextAction(str, Enum):
    ITERATE_SAME_IDEA = "iterate_same_idea"
    TRY_ALTERNATIVE_VARIANT = "try_alternative_variant"
    TRY_ALTERNATIVE_IDEA = "try_alternative_idea"
    INVESTIGATE_FAILURE = "investigate_failure"
    PUBLISH_OR_SHARE = "publish_or_share"

class NextRunProposal(BaseModel):
    proposal_id: str
    action: NextAction
    target_stage: str               # "3.2" | "3.4" | "3.5" | "3.8"
    requires_child_run: bool
    reason: str
    prerequisite_actions: list[str]
    evidence_refs: list[ArtifactRef]
```

路由规则：

```
瞬态执行失败 → 3.8 retry (同一 run)
环境/资产问题 → Environment/Asset resolution (同一 run)
patch/code 问题 → 3.6-3.7 重新审批 (同一 run)
实验设计问题 → 3.5 (同一 run)
换 Variant → 新 child run，从 3.4/3.5 开始
换 Idea → 新 child run；仅需要新论文时才回 3.2
```

```python
class Reflection(BaseModel):
    scientific_conclusion: ScientificConclusion
    recommended_action: NextAction
    key_learnings: list[str]
    unexpected_results: list[str]
    limitations: list[str]
    next_run_proposals: list[NextRunProposal]
```

Reflection 只能提建议，不能自动触发写操作或实验。

# 3. 报告生成：facts → LLM → validate

## 3.1 report_facts.json（确定性代码写入）

```json
{
  "run_id": "run_001",
  "idea_label": "Cross-Scale Attention",
  "baseline": "PatchCore",
  "dataset": "MVTec bottle",
  "scientific_conclusion": "beneficial",
  "comparisons": [...],
  "validity_interpretation": {...},
  "failure_analysis": {...},
  "resource_comparison": {...},
  "artifact_registry": [
    {"path": "attempt_01/metrics.json", "sha256": "abc..."},
    {"path": "attempt_02/metrics.json", "sha256": "def..."}
  ]
}
```

所有数值、SHA、状态、结论都是**确定性代码产生**的。

## 3.2 final_report.md（LLM 基于 facts 生成）

LLM 的输入只有 `report_facts.json`。LLM 负责：
- 结构排布（7 节）
- 可读性
- 重复内容合并
- 自然语言解释（例如"为什么 beneficial"）

LLM 不得修改任何数值、状态、结论或 SHA。

# 4. final_report.md 结构

```markdown
# AutoAD-Researcher 实验报告

## 1. 实验概要
Run ID, Idea, Baseline, Dataset, Metrics, 执行时间, 整体结论

## 2. Idea 与迁移设计
Idea 来源, 兼容性判断, 选择的 Variant, 预期效果

## 3. 执行过程
ExecutionUnit 概览, 各 attempt 状态, Reproducibility

## 4. 结果对比
指标对比表（含 comparison_status），方向分析，资源开销

## 5. 科学有效性
Validity 检查结果，评价协议一致性，局限

## 6. 结论
ScientificConclusion，结论充分性，NextRunProposal

## 7. 附录
Artifact 清单，SHA256，Evidence Index
```
