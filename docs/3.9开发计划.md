# 2. 分析维度

## 2.1 Baseline vs Variant 对比

```python
class MetricComparisonEntry(BaseModel):
    metric_name: str
    direction: Literal["maximize", "minimize"]
    baseline_value: float
    variant_value: float
    delta: float
    delta_pct: float
    improved: bool
    significant: bool  # 基于 AutoSOTA 的阈值：≥3%

class ComparisonReport(BaseModel):
    entries: list[MetricComparisonEntry]
    improved_count: int
    degraded_count: int
    unchanged_count: int
    summary: str  # LLM 生成
```

## 2.2 指标方向检查

```
direction = maximize → variant > baseline → improved
direction = minimize → variant < baseline → improved
变化在 tolerance 内 → unchanged
全部 unchanged → 实验无区分度 → inconclusive
```

## 2.3 评价协议一致性

```
baseline eval SHA == variant eval SHA → consistent
否则 → evaluation_protocol_diverged → validity 降级
```

## 2.4 运行失败归因

```python
class FailureCategory(str, Enum):
    PREFLIGHT_FAILURE = "preflight_failure"
    EXECUTION_CRASH = "execution_crash"
    TIMEOUT = "timeout"
    METRIC_PARSE_FAILURE = "metric_parse_failure"
    REPRODUCIBILITY_FAILURE = "reproducibility_failure"
    VALIDITY_FAILURE = "validity_failure"

class FailureAnalysis(BaseModel):
    attempt_id: str
    category: FailureCategory
    root_cause_summary: str
    actionable: bool
    suggested_fix: str | None
```

## 2.5 资源成本比较

```python
class ResourceComparison(BaseModel):
    baseline_runtime: float | None
    variant_runtime: float | None
    overhead_pct: float | None
    within_budget: bool
```

## 2.6 Idea 支持度判定

| 结果 | 判定 |
|------|------|
| 所有目标指标 improved | strongly_supported |
| 大部分 improved，个别 unchanged | partially_supported |
| 全部 unchanged | inconclusive |
| 有 degraded | not_supported |
| 实验失败 | cannot_judge |

## 2.7 结论充分性

| 条件 | 判定 |
|------|------|
| 2 attempt 成功 + reproducible + valid + metrics passed | sufficient |
| 任一失败 | insufficient |
| 全部 unchanged | insufficient（效应太小） |
| 部分改进 + 部分 unchanged | partial |

# 3. 输出

| 文件 | 内容 |
|------|------|
| `comparison_report.json` | baseline vs Variant 指标对比（emoji 编码） |
| `scientific_validity_report.json` | 含 3.8 validity + 3.9 语义分析 |
| `failure_analysis.json` | 如果失败，归因 + 可操作建议 |
| `resource_comparison.json` | 成本对比 |
| `reflection.json` | 经验总结 + 下一步建议 |
| `final_report.md` | 人类可读完整报告（7 节） |

## 3.1 Reflection

```python
class Reflection(BaseModel):
    idea_supported: Literal[
        "strongly_supported", "partially_supported",
        "not_supported", "inconclusive", "cannot_judge"
    ]
    conclusion_sufficient: bool
    key_learnings: list[str]
    unexpected_results: list[str]
    limitations: list[str]
    next_steps: list[str]
    recommended_actions: list[Literal[
        "iterate_on_current_idea",
        "try_alternative_variant",
        "try_alternative_idea",
        "investigate_failure",
        "publish_or_share",
    ]]
```

## 3.2 final_report.md 结构

```markdown
# AutoAD-Researcher 实验报告

## 1. 实验概要（Run ID, Idea, Baseline, Dataset, Metrics, 时间）

## 2. Idea 与迁移设计（来源、兼容性、选择的 Variant、预期效果）

## 3. 执行过程（Attempt 01/02、环境指纹、Reproducibility）

## 4. 结果对比（指标对比表、方向分析、资源开销）

## 5. 科学有效性（Validity checks、评价一致性、数据泄漏、局限）

## 6. 结论（Idea 是否支持、结论是否充分、下一步）

## 7. 附录（artifact 清单、SHA256、Evidence Index）
```
