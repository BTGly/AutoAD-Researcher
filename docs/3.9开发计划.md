> 文档状态：修订稿 v2.1（收口 4 项剩余 P0）
> 更新日期：2026-06-18
> 上游：3.8 ExperimentExecutionHandoff + 3.5 StatisticalAnalysisPlan
> 设计原则：**逐 Variant 出结论，不压成单个 run_conclusion。报告先生成事实（确定性），再 LLM 润色。Reflection 只提议不执行。**

# 0. 阶段定义

```
ExperimentExecutionHandoff (3.8)
+ StatisticalAnalysisPlan (3.5)
+ SharedExperimentProtocol (3.5, 直接引用，不经 3.8 转述)
    ↓
Evidence Completeness Assessment (per-variant seed pair 覆盖)
    ↓
Per-seed / per-variant / per-metric 观测表
    ↓
执行 StatisticalAnalysisPlan 决策规则
    ↓
逐 Variant ScientificConclusion (复用 3.5 枚举)
    ↓
Idea-support 确定性映射
    ↓
ValidityInterpretation (聚合 3.0 reports，不改写，不改名)
    ↓
FailureHypothesis (含 evidence)
    ↓
ResourceComparison (允许 not_available)
    ↓
Reflection + NextRunProposal
    ↓
report_facts.json (确定性)
    ↓
final_report.md (LLM 基于 facts)
    ↓
ReportFactValidator (正向+反向)
```

# 1. 核心模型

## 1.1 观测层

```python
class MetricObservationKey(BaseModel):
    variant_id: str | None
    metric_name: str
    dataset_row: str
    seed: int

class PerObservationMetric(BaseModel):
    key: MetricObservationKey
    unit_id: str
    parsed_metric_ref: ArtifactRef
    value: float | None
    parse_status: Literal["parsed", "missing", "invalid"]
    source_sha256: str
```

## 1.2 聚合对比层

```python
class AggregatedMetricComparison(BaseModel):
    variant_id: str
    metric_name: str
    dataset_row: str

    baseline_values: list[float]
    variant_values: list[float]
    paired_count: int

    aggregation_method: str
    baseline_aggregate: float | None
    variant_aggregate: float | None
    dispersion: float | None

    effect_size: float | None
    relative_change_pct: float | None

    comparison_status: Literal[
        "comparable",
        "missing_baseline", "missing_variant",
        "invalid_baseline", "invalid_variant",
        "insufficient_pairs",
    ]
```

## 1.3 EvidenceSufficiency（per-variant seed pair 覆盖）

```python
class EvidenceSufficiency(BaseModel):
    variant_id: str
    required_seed_ids: list[int]
    completed_valid_seed_pairs: list[int]
    missing_seed_ids: list[int]
    invalid_seed_ids: list[int]

    primary_metric_coverage: float
    evaluation_protocol_consistent: bool
    sufficient_for_decision: bool
    blocking_reasons: list[str]
```

有效 pair = baseline 与 Variant 在同一 seed、都 execution success、metrics parsed、validity valid、protocol identity 一致。

## 1.4 逐 Variant 科研结论

```python
class VariantScientificConclusion(BaseModel):
    variant_id: str
    conclusion: ScientificConclusion  # 3.5 枚举
    matched_rule_id: str
    completed_seed_pairs: list[int]
    missing_seed_pairs: list[int]
    evidence_refs: list[ArtifactRef]

class RunConclusionSummary(BaseModel):
    per_variant: list[VariantScientificConclusion]
    best_supported_variant_ids: list[str]
    incomplete_variant_ids: list[str]
```

不能把三个 Variant 的不同结论压缩成单一 `run_conclusion=mixed`。

## 1.5 Idea-support 确定性映射

```
单 Variant:
  beneficial              → supported
  practically_equivalent  → not_demonstrated
  mixed                   → partially_supported_or_tradeoff
  worse                   → not_supported
  incomplete              → cannot_judge

多 Variant:
  至少一个 beneficial → idea_supported_by_at_least_one_variant
  全部 worse           → idea_not_supported_by_tested_variants
  全部 incomplete      → cannot_judge
  一个 beneficial、一个 worse → implementation_sensitive
```

LLM 只能撰写解释文本，不能修改这里的映射结果。

## 1.6 Validity 聚合层（不改名、不改写）

```python
class ValidityInterpretation(BaseModel):
    per_unit_validity_refs: list[ArtifactRef]
    valid_unit_ids: list[str]
    invalid_unit_ids: list[str]
    insufficient_evidence_unit_ids: list[str]

    protocol_consistency: Literal["consistent", "diverged"]
    protocol_divergence_details: list[str]
    evaluation_contract_fingerprint: str

    conclusion_eligibility_by_variant: dict[str, bool]
    evidence_refs: list[ArtifactRef]
```

输出命名为 `scientific_validity_aggregate.json`，不与 3.0 每次 attempt 的 `scientific_validity_report.json` 同名。

## 1.7 资源比较

```python
class ResourceComparison(BaseModel):
    variant_id: str
    dimension: Literal["wall_time", "peak_rss", "peak_gpu_memory", "gpu_seconds"]
    status: Literal["comparable", "missing_baseline", "missing_variant", "incompatible_measurement"]
    baseline_value: float | None
    variant_value: float | None
    delta: float | None

class ResourceComparisonReport(BaseModel):
    per_variant: dict[str, list[ResourceComparison]]
    within_budget: bool
    measurement_quality: Literal["complete", "partial", "unavailable"]
```

## 1.8 失败分析

```python
class FailureHypothesis(BaseModel):
    hypothesis_id: str
    unit_id: str
    category: FailureCategory
    claim: str
    status: Literal["confirmed", "inferred", "unknown"]
    confidence: Literal["high", "medium", "low"]
    evidence_refs: list[ArtifactRef]
    falsification_steps: list[str]
    alternative_hypotheses: list[str]

class FailureAnalysis(BaseModel):
    failed_unit_ids: list[str]
    hypotheses: list[FailureHypothesis]
    execution_errors: list[str]       # 确定性
    inferred_causes: list[str]        # LLM 辅助
    actionable_failures: list[str]
    blocked_failures: list[str]

    # 建议路由（确定性映射）
    suggested_routing: dict[str, Literal[
        "3.8_retry",
        "environment_asset_resolution",
        "3.6_3.7_reapproval",
        "3.5_revision",
        "child_run",
        "stop",
    ]]
```

## 1.9 Reflection + NextRunProposal

```python
class NextRunProposal(BaseModel):
    proposal_id: str
    action: NextAction
    target_stage: str
    requires_child_run: bool
    reason: str
    prerequisite_actions: list[str]
    evidence_refs: list[ArtifactRef]

class Reflection(BaseModel):
    idea_support_summary: str  # LLM 撰写
    per_variant_conclusions: dict[str, ScientificConclusion]  # 确定性
    recommended_action: NextAction
    key_learnings: list[str]
    unexpected_results: list[str]
    limitations: list[str]
    next_run_proposals: list[NextRunProposal]
```

只提议，不自动触发写操作或实验。

## 1.10 报告生成

```python
class ReportFacts(BaseModel):
    run_id: str
    idea_label: str
    baseline: str
    dataset: str
    per_variant_conclusions: dict[str, ScientificConclusion]
    idea_support: str
    comparisons: list[AggregatedMetricComparison]
    validity_aggregate: ValidityInterpretation
    failure_analysis: FailureAnalysis | None
    resource_comparison: ResourceComparisonReport | None
    artifact_registry: list[ArtifactRef]
    evidence_sufficiency: dict[str, EvidenceSufficiency]

class ReportFactValidation(BaseModel):
    passed: bool
    issues: list[str]  # 篡改数值、遗漏 invalidity、省略结论等
```

# 3. 测试（追加）

```
[ ] 三个 Variant → 三个独立 VariantScientificConclusion
[ ] baseline/variant 缺 seed → 该 pair 不计入 EvidenceSufficiency
[ ] validity invalid 的 seed 不进入聚合
[ ] practically_equivalent → not_demonstrated（不映射为 supported）
[ ] beneficial + worse → implementation_sensitive
[ ] 全部失败 → INCOMPLETE final report，包含 FailureAnalysis
[ ] reused baseline 在报告中明确标记 baseline_source=reused
[ ] 缺 resource telemetry → not_comparable，不编造成本
[ ] final_report 遗漏 invalidity → ReportFactValidator 拒绝
[ ] final_report 篡改数值 → ReportFactValidator 拒绝
[ ] Reflection 只生成 proposal，不自动创建 child run
[ ] 报告输出 scientific_validity_aggregate.json（不是 scientific_validity_report.json 同名）
```
