## 0.1 纵向闭环（loop）

3.9 的 `reflection.json.recommended_actions` 决定下一步：

```
iterate_on_current_idea    → 回到 3.5（同 Idea 新实验计划）
try_alternative_variant    → 回到 3.4（选不同 Variant）
try_alternative_idea       → 回到 3.2/3.4（新 Idea）
investigate_failure        → 回到 3.8（修复后重跑）
publish_or_share           → 闭环结束
```

这就是"真实纵向闭环"——不是一次跑完就结束，而是 3.9 的结论可以驱动新一轮迭代。

## 1.1 AI-Scientist `perform_experiments.py` — 实验执行闭环

```
references/research-automation/AI-Scientist/ai_scientist/perform_experiments.py (lines 116–166)
```

**直接借鉴 3 个模式**：

| 模式 | 原文 | 3.8 映射 |
|------|------|---------|
| 指标解析格式 | `final_info.json` with `{metric: {"means": {...}, "stderrs": {...}}}` | 3.8 `metrics.json` 的 canonical schema |
| Timeout + stderr 截断 | `subprocess.run(timeout=7200)`, stderr > 1500 chars 自动截断 | `attempt_manager` 的执行安全 |
| `notes.txt` 累积 | 每个 run 结果追加到 `notes.txt`，writeup 阶段只消费文件 | 3.8 → 3.9 evidence 链 |

```python
# AI-Scientist 核心执行循环（简化）
coder_out = coder.run(next_prompt)        # LLM 写 experiment.py
return_code, next_prompt = run_experiment(folder, run)  # subprocess.run
if return_code == 0:
    results = json.load(final_info.json)   # {metric: {"means": {...}}}
    run += 1
    current_iter = 0
else:
    current_iter += 1                      # 失败 → LLM 修复
```

## 1.4 autoresearch `program.md` — 单文件修改 + 固定时间预算 + TSV 日志

```
references/research-automation/autoresearch/program.md
```

**核心模式**（第 54-89 行）：

# 2. 需要新增

## 2.1 ExperimentPlan → ExperimentCommandPlan 桥

`ExperimentPlan`（3.5 产出）→ `ExperimentCommandPlan`（3.0 已有）：

```
ExperimentPlan
  → resolve baseline/variant command
  → inject patch location / dataset path / metric specs
  → build ExperimentCommandPlan
```

## 2.2 AttemptManager

```
AttemptManager
  → 拍 repository_before.json（指纹快照）
  → 调用 execute_experiment_attempt()
  → 拍 repository_after.json
  → 收集 stdout.log / stderr.log
  → 调用 parse_metrics() → metrics.json（对接 AI-Scientist 的 final_info.json 格式）
  → 调用 validate_scientific_contract() → validity_report.json
  → Hook chain: on_attempt_completed()
```

## 2.3 模型

```python
class AttemptSummary(BaseModel):
    attempt_id: str  # "attempt_01", "attempt_02"
    status: Literal[
        "preflight_failed", "execution_failed",
        "metric_parse_failed", "validity_failed", "success"
    ]
    started_at: datetime
    completed_at: datetime | None
    execution_result_sha256: str | None
    metrics_report_sha256: str | None
    validity_report_sha256: str | None

class ExecutionManifest(BaseModel):
    run_id: str
    experiment_plan_sha256: str
    patch_sha256: str
    command_plan: ExperimentCommandPlan
    preflight_report_sha256: str
    attempts: list[AttemptSummary]
    reproducibility_report_sha256: str | None
```

## 2.4 3.9 Handoff

```python
class ExperimentExecutionHandoff(BaseModel):

    execution_manifest: ExecutionManifest
    baseline_metrics: dict[str, float] | None
    variant_metrics: dict[str, float] | None
```
