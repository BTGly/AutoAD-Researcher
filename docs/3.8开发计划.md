# 0. 阶段定义

```
ExperimentPlannerHandoff (3.5)  +  PatchRunnerHandoff (3.7)
    ↓
Runner Intake Validator  → RunnerIntakeReport (status 由 Validator 产生)
    ↓
Matrix → ExecutionUnitPlans (含依赖 DAG、artifact binding)
    ↓
每个 ExecutionUnit:
  Environment/Asset resolution
  → ExperimentCommandPlan (3.0)
  → Preflight (3.0)
  → Attempt 01
  → ResourceUsageReport
  → MetricsReport (3.0 ParsedMetric 格式)
  → ScientificValidityReport (3.0)
  → RetryDecision (仅瞬态错误，command/input SHA 不变)
  → Attempt 02 (仅 retry_same_command)
    ↓
ExecutionManifest
    ↓
ExperimentExecutionHandoff → 3.9
```

# 1. 核心模型

## 1.1 WorkspaceExecutionRef（多 workspace 身份）

3.6-3.7 已支持 workspace_01→VariantA/B, workspace_02→VariantC。Baseline 绑定未打 patch 的 workspace：

```python
class WorkspaceExecutionRef(BaseModel):
    workspace_id: str
    subject_type: Literal["baseline", "variant"]
    variant_ids: list[str]  # baseline 时为 []
    repository_fingerprint: str
    patch_diff_sha256: str | None  # baseline 时为 None
    local_validation_report_sha256: str
    patch_runner_handoff_sha256: str
```

## 1.2 RunnerIntakeRequest / RunnerIntakeReport（status 分离）

```python
class RunnerIntakeRequest(BaseModel):
    experiment_planner_handoff_sha256: str
    experiment_matrix_sha256: str
    shared_protocol_fingerprint: str
    statistical_analysis_plan_sha256: str
    operational_guard_policy_sha256: str
    workspace_refs: list[WorkspaceExecutionRef]
    # 不含 intake_status —— 由 Validator 产生

class IntakeCheck(BaseModel):
    check_id: str
    passed: bool
    description: str

class RunnerIntakeReport(BaseModel):
    status: Literal["eligible", "blocked", "needs_revalidation"]
    checks: list[IntakeCheck]
    report_sha256: str
```

Validator 强制规则：
- baseline workspace 必须存在且 `subject_type=baseline`
- 每个 variant 有对应 workspace，fingerprint 不串用
- 所有 SHA 可解析

## 1.3 ExecutionUnitPlan（含依赖 DAG 和 artifact binding）

```python
class ArtifactBinding(BaseModel):
    role: str  # "fitted_weights", "smoke_log", "full_metrics"
    unit_id: str  # 由哪个 unit 产生
    expected_sha256: str | None

class ExecutionUnitPlan(BaseModel):
    unit_id: str
    matrix_entry_id: str

    subject_type: Literal["baseline", "variant"]
    variant_id: str | None
    trial_phase: str  # 复用 3.5 的 TrialPhase enum，不手写
    seed: int | None  # baseline/full/variant per-seed 必填；shared fit 为 None

    workspace_id: str
    command_plan: ExperimentCommandPlan  # 3.0
    input_refs: ExperimentInputRefs      # 3.0
    metric_parse_specs: list[MetricParseSpec]  # 3.0
    guard_policy_entry_id: str

    # 依赖 DAG
    depends_on_unit_ids: list[str]
    required_input_artifacts: list[ArtifactBinding]
    expected_output_artifacts: list[str]  # artifact roles produced

class ExecutionUnitStatus(str, Enum):
    INTAKE_VALIDATING = "intake_validating"
    ENVIRONMENT_RESOLVING = "environment_resolving"
    ASSETS_RESOLVING = "assets_resolving"
    COMMAND_BINDING = "command_binding"
    PREFLIGHT_RUNNING = "preflight_running"
    EXECUTION_RUNNING = "execution_running"
    METRIC_PARSING = "metric_parsing"
    VALIDITY_CHECKING = "validity_checking"
    RETRY_DECISION_PENDING = "retry_decision_pending"
    BLOCKED_UPSTREAM_FAILURE = "blocked_upstream_failure"
    UNIT_COMPLETED = "unit_completed"
    UNIT_FAILED = "unit_failed"
    UNIT_BLOCKED = "unit_blocked"
```

依赖调度规则：
- 依赖 unit 未成功 → `BLOCKED_UPSTREAM_FAILURE`
- 依赖 artifact SHA 不匹配 → `preflight_failed`
- 不得继续执行空输入的 full trial

## 1.4 AttemptRecord（只存 refs，状态由 refs 确定性派生）

```python
class AttemptRecord(BaseModel):
    attempt_id: str
    attempt_index: int

    execution_result_ref: ArtifactRef | None
    metrics_report_ref: ArtifactRef | None
    validity_report_ref: ArtifactRef | None
    resource_usage_ref: ArtifactRef | None

    started_at: datetime
    completed_at: datetime | None

    # 缓存状态 —— 由 Validator 从 refs 重算，不一致则拒绝
    _cached_execution_status: str | None = PrivateAttr(default=None)
    _cached_metric_status: str | None = PrivateAttr(default=None)
    _cached_validity_status: str | None = PrivateAttr(default=None)
```

不复制自由字段。Validator 必须从 ref 指向的 artifact 重算状态并校验一致性。

## 1.5 RetryDecision（含不可变性规则）

```python
class RetryDecision(BaseModel):
    unit_id: str
    failed_attempt_id: str

    decision: Literal[
        "retry_same_command",
        "do_not_retry",
        "return_to_3_5",
        "return_to_3_6_3_7",
        "blocked",
    ]
    reason_code: str
    evidence_refs: list[ArtifactRef]
```

确定性规则：
```
attempt_02.command_sha256 == attempt_01.command_sha256  → 才允许 retry
attempt_02.input_refs == attempt_01.input_refs          → 才允许 retry
attempt_02.workspace fingerprint == attempt_01          → 才允许 retry
任一不匹配 → do_not_retry (这是新 unit，不是 retry)

瞬时进程错误/资源抢占 → retry_same_command
metric_parse_failed → 不可 retry（产出格式问题）
validity invalid → 不可 retry（不能"刷成 valid"）
repository mutation → 不可 retry
代码/patch 错误 → return_to_3_6_3_7
实验设计缺陷 → return_to_3_5
```

class ExecutionUnitRecord(BaseModel):

class ExperimentExecutionHandoff(BaseModel):

# 2. 状态机与 Resume

```
（同 v2，增加 BLOCKED_UPSTREAM_FAILURE）

Resume 规则：
- unit_completed + artifact SHA 完整 → skip
- unit_running 无终态 → 标记 interrupted，不覆盖旧 attempt 目录
- attempt 目录已存在 → 创建新 attempt index 或恢复，不删除旧 evidence
- 上游 SHA 变化 → 全部 command plans 失效
```

# 4. 测试（追加）

```
[ ] baseline 与 Variant 映射到不同 workspace，fingerprint 不串用
[ ] baseline workspace 无 patch_diff
[ ] RunnerIntakeReport.status 由 Validator 产生，不由 request 输入
[ ] 每个 Matrix entry 恰好生成一个 unit，无遗漏无重复
[ ] fit dependency 失败 → full unit BLOCKED_UPSTREAM_FAILURE
[ ] shared fit unit seed=None；baseline/full seed 必填
[ ] attempt_02 command/input/workspace SHA 与 attempt_01 一致
[ ] metric_parse_failed / validity invalid 不触发 retry
[ ] interrupted attempt resume 不覆盖旧目录
[ ] resource unavailable → ResourceUsageReport(status=not_available)
[ ] AttemptRecord 缓存状态与 ref 派生状态不一致 → Validator 拒绝
[ ] dependency artifact SHA 不匹配 → preflight_failed
```
