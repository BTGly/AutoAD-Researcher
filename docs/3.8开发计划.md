# 0. 阶段定义

```
ExperimentPlannerHandoff (3.5)  +  PatchRunnerHandoff (3.7)
    ↓
Runner Intake Validator
    ↓
Matrix → ExecutionUnitPlans (一层：规划到执行)
    ↓
每个 ExecutionUnit:
  Environment/Asset resolution
  → immutable ExperimentCommandPlan (3.0)
  → Preflight (3.0)
  → Attempt 01 → RetryDecision → Attempt 02 (二层：操作重试)
    ↓
  → MetricsReport (3.0 analysis/metrics)
  → ScientificValidityReport (3.0 supervisor/validity)
    ↓
ExecutionManifest (三层：汇总)
    ↓
ExperimentExecutionHandoff → 3.9
```

## 0.1 三层模型

```
ExperimentMatrix (来自 3.5)
  ├── ExecutionUnit 1 (baseline, seed=42)
  │     ├── Attempt 01 (操作执行)
  │     └── Attempt 02 (仅当 RetryDecision=retry_same_command)
  ├── ExecutionUnit 2 (Variant A, fit)
  │     └── Attempt 01
  ├── ExecutionUnit 3 (Variant A, smoke, seed=42)
  ├── ExecutionUnit 4 (Variant A, full, seed=42)
  └── ...
```

- **ExecutionUnit** = 3.5 Matrix 中一个 entry + 一个 seed 的不可变命令
- **Attempt** = 同一 command 因超时/瞬态错误的操作重试
- **Seed** = 3.5 预注册的科研重复（在 Matrix 层，不在 Attempt 层）

## 0.2 不做

- 重新实现 Runner（已有 3.0 `runner/executor.py`）
- 重新实现 Metrics Parser（已有 3.0 `analysis/metrics.py`，量纲 `ParsedMetric`/`MetricsReport`）
- 重新实现 Validity Supervisor（已有 3.0 `supervisor/validity.py`，量纲 `ScientificValidityReport`）
- 修改 3.5 的 ExperimentMatrix 结构
- 把 Attempt 当作 Seed（3.9 才做 seed 聚合）
- 失败后 LLM 改代码直接重跑（改代码必须回到 3.6→3.7）

## 2.5 RetryDecision

```python
class RetryDecision(BaseModel):
    unit_id: str
    failed_attempt_id: str

    decision: Literal[
        "retry_same_command",       # 瞬时错误 → 重试相同命令
        "do_not_retry",             # 不重试，标记 unit_failed
        "return_to_3_5",            # 实验设计问题 → 回 3.5
        "return_to_3_6_3_7",        # patch/code 问题 → 回 3.6-3.7
        "blocked",                  # 无法解决
    ]
    reason_code: str
    evidence_refs: list[ArtifactRef]

# 确定性规则:
# 瞬时进程错误/资源抢占 → retry_same_command
# command SHA / repo fingerprint / asset hash 不一致 → do_not_retry → 重新 preflight
# 代码错误/patch 错误 → return_to_3_6_3_7 → 重新审批
# 实验设计缺陷 → return_to_3_5
# Validity invalid → 不得通过重复运行"刷成 valid"
```

# 3. 状态机与 Resume

```
intake_validating
  → eligible → environment_resolving
  → blocked  → unit_blocked (end)

environment_resolving → assets_resolving
  → failed → unit_blocked

assets_resolving → command_binding
  → failed → unit_blocked

command_binding → preflight_running
  → failed → retry_decision_pending

preflight_running → execution_running
  → failed → retry_decision_pending

execution_running → metric_parsing
  → failed → retry_decision_pending

metric_parsing → validity_checking
  → failed → unit_failed (不重试——metric 解析失败是产出的格式问题，不是瞬态)

validity_checking → unit_completed / unit_failed

retry_decision_pending
  → retry_same_command → execution_running (attempt_02)
  → do_not_retry → unit_failed
  → return_to_3_5 / return_to_3_6_3_7 → unit_blocked

all units terminal → handoff_ready → 3.9
```

Resume 规则：
- attempt 目录存在时不覆盖
- 同一 command SHA 不被静默替换
- 已完成 unit 可安全 resume（跳过）
- 上游 SHA 变化 → 已生成 command plans 全部失效
- 失败 unit 是否允许 retry 由 RetryDecision 决定
