# 0. 阶段定义

3.8 负责将 Step 3.5 的 `ExperimentPlan` 和 Step 3.6 的 patch 转换为**不可变的命令计划**，通过已有的 3.0 Runner 执行，解析产物，验证有效性，并进行跨 attempt 对比。

```
ExperimentPlan + PatchApproval
    ↓
Immutable Command Plan (ExperimentCommandPlan)
    ↓
Environment/Asset Resolution (reuse 3.0)
    ↓
Preflight (reuse 3.0 benchmarks/preflight)
    ↓
Attempt 01 → execute (reuse runner/executor)
    ↓
Parse Metrics (reuse analysis/metrics)
    ↓
Validate (reuse supervisor/validity)
    ↓
Attempt 02 (if needed for reproducibility)
    ↓
Compare Attempts (reuse analysis/reproducibility)
    ↓
输出给 Step 3.9 Results Analysis
```

- 重新实现 Runner（已有 `runner/executor.py`）

# 2. 需要新增的内容

## 2.1 ExperimentPlan → Immutable Command Plan 桥

`ExperimentPlan`（3.5 产出）是**计划层**artifact，描述"要做什么"。`ExperimentCommandPlan`（3.0 已有）是**执行层**artifact，描述"shell 层面如何运行"。

需要桥接逻辑将 ExperimentPlan 转换为一个或多个 `ExperimentCommandPlan`：

```
ExperimentPlan
  → resolve baseline command (from environment)
  → inject patch location
  → resolve dataset path
  → resolve metric specs
  → build ExperimentCommandPlan
```

## 2.2 Attempt Manager

```
AttemptManager
  → 生成 attempt_01 目录
  → 执行前拍 repository_before.json（指纹快照）
  → 调用 execute_experiment_attempt()
  → 执行后拍 repository_after.json
  → 收集 stdout.log / stderr.log
  → 调用 parse_metrics() → metrics.json
  → 调用 validate_scientific_contract() → validity_report.json
  → 如果需要 attempt_02：重复上述流程
  → 调用 compare_attempts() → reproducibility_report.json
```

## 2.3 Execution Manifest

汇总所有 attempt 的元数据：

```python
class ExecutionManifest(BaseModel):
    run_id: str
    experiment_plan_sha256: str
    patch_sha256: str
    approval_decision_id: str | None

    command_plan: ExperimentCommandPlan
    environment_plan_sha256: str
    asset_manifest_sha256: str | None

    preflight_report_sha256: str

    attempts: list[AttemptSummary]
    reproducibility_report_sha256: str | None

class AttemptSummary(BaseModel):
    attempt_id: str  # "attempt_01", "attempt_02"
    status: Literal["preflight_failed", "execution_failed", "metric_parse_failed", "validity_failed", "success"]
    started_at: datetime
    completed_at: datetime | None
    execution_result_sha256: str | None
    metrics_report_sha256: str | None
    validity_report_sha256: str | None
    repository_before_sha256: str
    repository_after_sha256: str | None
```

## 2.4 Step 3.9 Handoff

```python
class ExperimentExecutionHandoff(BaseModel):
    run_id: str
    execution_manifest: ExecutionManifest

    # 每个 attempt 的指针
    attempt_results: dict[str, dict]  # attempt_id → paths to all artifacts

    # 汇总
    metrics: list[ParsedMetric]
    validity_checks: list[ValidityCheck]
    reproducibility: ReproducibilityReport | None

    # 3.9 需要的关键信息
    baseline_metrics: dict[str, float] | None  # attempt_01
    variant_metrics: dict[str, float] | None  # attempt_02

    next_stage: Literal["3.9_results_analysis"]
```
