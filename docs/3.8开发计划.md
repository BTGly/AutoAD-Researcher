# 0. 阶段定义

```
ExperimentPlannerHandoff (3.5) + PatchRunnerHandoff v2 (3.7, schema_version=2)
    ↓
Runner Intake Validator → RunnerIntakeReport
    ↓
Matrix → Immutable ExecutionUnitPlans
    ↓
每个 ExecutionUnit:
  → ExperimentCommandPlan (3.0) → Preflight (3.0)
  → Attempt 01: AttemptIdentitySnapshot → ResourceUsageReport →
    MetricsReport → ScientificValidityReport → RetryDecision
  → Attempt 02 (仅 retry_same_command 且 identity 匹配)
    ↓
ExecutionUnitRecord + ExecutionManifest
    ↓
ExperimentExecutionHandoff → 3.9
```

## 0.1 关键原则

- `ExecutionUnitPlan` 不可变，SHA 一经计算永不改变
- 运行时解析结果入 `AttemptRecord`，不入 Plan
- 计划态 binding 有唯一 `binding_id`
- Retry = 失败分类允许 + identity 匹配

# 2. WorkspaceExecutionRef + Intake

```python
class WorkspaceExecutionRef(BaseModel):
    workspace_id: str
    subject_type: Literal["baseline", "variant"]
    variant_ids: list[str]
    repository_fingerprint: str; repository_commit: str
    patch_diff_sha256: str | None
    local_validation_report_sha256: str | None
    patch_runner_handoff_ref: ArtifactReference | None

    @model_validator(mode="after")
    def _validate(self):
        if self.subject_type == "baseline":
            assert not self.variant_ids
            assert self.patch_diff_sha256 is None
            assert self.local_validation_report_sha256 is None
            assert self.patch_runner_handoff_ref is None
        else:
            assert self.variant_ids
            assert self.patch_diff_sha256 is not None
            assert self.local_validation_report_sha256 is not None
            assert self.patch_runner_handoff_ref is not None
        return self

class RunnerIntakeRequest(BaseModel):
    experiment_planner_handoff_sha256: str; experiment_matrix_sha256: str
    shared_protocol_fingerprint: str; statistical_analysis_plan_sha256: str
    operational_guard_policy_sha256: str; workspace_refs: list[WorkspaceExecutionRef]

class RunnerIntakeReport(BaseModel):
    status: Literal["eligible", "blocked", "needs_revalidation"]
    checks: list[IntakeCheck]; report_sha256: str
```

整体 Validator：恰好 1 baseline workspace；workspace_id 不重复；variant 不跨 workspace 重复。

# 5. 参考实现启发与设计溯源

> **非规范性设计溯源章节。** 正式行为以本文 schemas、validators 和状态机为准。参考项目不得覆盖 AutoAD 的审批、审计和预注册规则。

## 5.1 映射总表

| 参考项目 | 借鉴模式 | AutoAD 落点 | 采用/改造/拒绝 |
|---------|---------|------------|--------------|
| AI-Scientist `perform_experiments.py` | 实验进程执行、timeout、stdout/stderr 留存 | `ExperimentCommandPlan` + `AttemptRecord` + 执行目录 | **改造**：`final_info.json` 仅作为 `MetricsReport` adapter 的输入格式参考。timeout/stderr 归入 Attempt execution evidence 与 FailureAnalysis 输入。wall time/memory/GPU telemetry 由独立的 `ResourceUsageReport` 记录，不混入 timeout/stderr |
| AI-Scientist `perform_experiments.py` | 实验结束后的结构化结果摘要 | `MetricsReport` + `ExecutionManifest` | **改造**：AI-Scientist 的 `{metric: {"means": {...}}}` 格式仅作为 adapter 输入参考，AutoAD 正式合同是 `ParsedMetric` 列表 |
| AI-Scientist `perform_experiments.py` | 失败也保留执行证据 | failed attempt artifact 完整留存 | **采用** |
| AI-Scientist `perform_experiments.py` | `notes.txt` 累积模式——每个 run 结果追加写入 | `ExecutionManifest` 由所有 attempt 的 evidence 确定性导出 | **改造**：不依赖 LLM 记忆，each-attempt artifact 落盘后再汇总 |
| SWE-Agent hook 系统 | `on_instance_completed` 注入 validity/safety check | `AttemptLifecycleHook`：execution completed → metrics parser → validity evaluator → artifact finalizer | **改造**：hook 只能生成派生 artifact，不能绕过审批修改仓库，validity 最终状态仍由确定性 Validator 决定 |
| SWE-Agent `should_skip/redo` | 检测 early_exit → 自动清理重跑 | `RetryDecision` + identity 匹配检查 | **改造**：增加失败分类（瞬态/vs 代码 bug/vs design），不是所有失败都可以 retry |
| SWE-Agent `compare_runs.py` | emoji 编码的基线对比 | `PairedMetricObservation` + `AggregatedMetricComparison` | **改造**：emoji table 只是 presentation 层，canonical data 仍是结构化 metric comparison（在 3.9） |
| autoresearch `program.md` | 固定时间预算 + 单一可修改文件 | `ExperimentCommandPlan.timeout` + `OperationalGuardPolicy` | **采用** |
| autoresearch `program.md` | `results.tsv` 持续追加的实验台账 | ExecutionManifest 的人类可读导出 | **改造**：`results.tsv` 不作为正式事实源，正式源仍为 JSON artifacts。内容由 `ExecutionManifest` 确定性导出 |
| autoresearch `program.md` | "NEVER STOP" 自主实验循环 | `RetryDecision` 驱动 attempt_02，不依赖人工中断 | **改造**：AutoAD 的循环有明确的终止条件（budget 耗尽、max_reanalysis_rounds、validity invalid 不可 retry） |
| AI-Scientist `launch_scientist.py` | 全生命周期 timing + 异常记录 | `ExecutionUnitRecord.started_at/completed_at` + `AttemptRecord` | **采用** |
| anomalib `Evaluator` | validation/test 指标采集生命周期 | `MetricParseSpec` → after-attempt hook | **改造**：不依赖 PyTorch Lightning hook，改为从 JSON/CSV 文件解析 |
| mlflow `EvaluationEntity` | 结构化结果模型（inputs/outputs/targets/metrics/error_code） | `AttemptRecord` 的多层 ref 结构 | **改造**：AutoAD 的 evidence 链要求每个 ref 都有 SHA，不直接保存 value |
| mini-swe-agent `AgentConfig` | 内置 step/cost/time 限制 | `OperationalGuardPolicy` + `RetryDecision` | **采用** |

## 5.2 明确拒绝的模式

| 模式 | 拒绝原因 |
|------|---------|
| LLM 自动修改代码后直接重跑（AI-Scientist） | 修改代码必须重新经过 3.6–3.7 审批 |
| 失败后绕过审批跑新实验（autoresearch "NEVER STOP" 全自由修改） | AutoAD 的实验循环受 `OperationalGuardPolicy` 和 `approved_patch` 约束 |
| 使用 review ensemble 覆盖统计结论（AI-Scientist `perform_review.py`） | 统计结论由 3.5 的 `StatisticalAnalysisPlan` 确定性地决定，不交给 LLM 投票 |
| 把展示表格作为 canonical 数据源（SWE-Agent emoji table） | 结构化 JSON artifacts 是正式事实源，展示层是派生产物 |
| AI-Scientist 的 `coder_prompt` 让 LLM 直接写 `experiment.py` | AutoAD 的 patch 由 3.6 生成，经 3.7 审批后 `git apply`，不是 LLM 直接写文件 |
