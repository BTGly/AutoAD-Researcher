## Scope

This addendum seals the generic Tool Foundation required by Step 3.1
Repository Intelligence. It is an additive 3.0-F seal and does not rewrite the
original Stage 3.0 final report.

The original Stage 3.0 report intentionally recorded
`tool_foundation_status: missing` because these generic tools did not exist at
the time of that seal. This addendum records the later implementation and
verification of those tools.

## Sealed Capabilities

This addendum covers:

- `ToolSpec`, `ToolResult`, `ToolContext`, and `ToolRegistry`
- deferred tool loading by stage
- layered `PermissionEngine` with allow / ask / deny decisions
- audited argv-based `ProcessTool` using `shell=False`
- workspace-scoped `filesystem_list`, `filesystem_read`, `filesystem_search`,
  and `filesystem_stat`
- stage-driven mandatory Skill loading with SHA audit
- `ActiveRepositoryContext` injection into tool context
- automatic filesystem evidence middleware for active repository reads/searches
- recorded WebSearch fixture provider
- secure WebFetch provider
- read-only GitHub metadata / commit / file-content provider

## Step 3.1 Entry Decision

The generic Tool Foundation is ready for Step 3.1 R4 and later integration
work. Step 3.1 still must implement its own repository-specific harness,
discovery, acquisition, attestation, analysis, validation, repair, handoff, CLI,
integration tests, documentation, and final seal.

Real external repository work remains prohibited until the Step 3.1 harness
supplies the required active repository context, acquisition policy, state
attestation, evidence validation, and audit trail for that stage.
