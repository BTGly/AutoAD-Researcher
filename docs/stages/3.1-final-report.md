## Scope

Stage 3.1 seals Repository Intelligence for the current AutoAD-Researcher
engineering baseline. The sealed path is deterministic and CI-safe: it uses
local Git fixtures, recorded/fake providers, scripted analysis behavior,
strict artifacts, deterministic validation, bounded repair, and GitHub Actions
verification.

This seal covers:

- Repository Intelligence contracts and control signals
- mandatory Repository Skills and Skill loading audit
- repository-aware evidence middleware
- harness stage entry, model routing, control-signal transition decisions, and
  resume fingerprinting
- deterministic discovery/resolution primitives
- controlled acquisition and repository state attestation
- read-only analysis agent skeleton
- seven formal Repository Intelligence artifacts
- deterministic evidence validator
- bounded repair loop
- EnvironmentPlan handoff
- Intent Clarifier handoff
- offline local-path CLI and resume behavior
- acquisition profile, security, budget, transition, repair, and integration
  fixtures
- public-repository-shaped CI fixtures covering two research repositories and
  one non-anomaly-detection repository
- operator-facing Repository Intelligence documentation

This seal does not claim:

- live WebSearch quality
- live LLM semantic quality
- live public GitHub clone reliability across the internet
- automatic repository modification
- dependency installation or repository code execution
- final environment decisions
- final benchmark validity for MVTec AD real data

Manual live provider evidence remains deferred to explicit manual integration
workflows with approved credentials, budgets, and audit retention. That
deferred evidence does not block this CI-sealed Stage 3.1 baseline.

- explicit GitHub URL skips WebSearch
