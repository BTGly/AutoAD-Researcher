### Findings

| Before | After | Why |
| --- | --- | --- |
| `.project-sidebar { transition: width ... }` | Keep the flex item at its rail width; animate an expanded inner layer with `transform` and `opacity` | Width is layout animation and can recalculate the whole workbench. The current mouse-only trigger also leaves keyboard and touch users with a collapsed label layer. |
| `animation: toastIn ...` | Transition `transform` and `opacity` from the current presentation value; retain the node through exit | Toasts can arrive repeatedly. Keyframes restart from zero and the current component unmounts immediately, so there is no symmetric exit or interruptible stack behavior. |
| Popover rendered with no motion and default origin | `scale(0.95)` + opacity, `transform-origin` anchored to the trigger, strong ease-out under 200ms | The menu should have a spatial relationship with its Plus or History trigger; `scale(0)` and center-origin motion are explicitly disallowed by the reference standards. |
| Global `button:active { opacity: 0.8 }` | Targeted `.app-button`, `.icon-button`, toolbar and task controls with `scale(0.97)` over 100–160ms | A press should acknowledge pointer-down immediately. Core navigation and high-frequency data controls remain visually stable. |
| `left: on ? 23 : 3; transition: left 0.2s` | Fixed thumb position plus `transform: translateX(...)` | `left` is a layout-affecting property; the interaction is a classic two-state compositor animation. |
| Every reduced-motion transition becomes `0.01ms` | Keep color/opacity feedback, remove translation/scale/overshoot, and add reduced-transparency and increased-contrast surfaces | Reduced motion is a gentler equivalent, not loss of all feedback. The Apple reference also treats transparency and contrast as independent preferences. |

### High-conviction opportunities

These are the only candidates that survive frequency, purpose, speed, and
function gates:

1. Sidebar expansion: spatial consistency and wayfinding; compositor-only
   movement, no content reflow, `--ease-out: cubic-bezier(0.23, 1, 0.32, 1)`.
2. Button press feedback: feedback; `scale(0.97)`, 100–160ms ease-out; excluded
   from the primary navigation rail and data rows.
3. Toast enter/exit: preventing a jarring change; transition `transform` and
   `opacity`, under 250ms, same edge in both directions.
4. Upload and Session popovers: spatial consistency; `scale(0.95)` plus opacity,
   trigger origin, 125–200ms ease-out.
5. Modal appearance: state indication and hierarchy; centered `scale(0.97)` plus
   scrim opacity, 200–300ms, with focus returned to the trigger on close.
6. Settings toggle thumb: state indication; `transform` under 200ms, no bounce.

### Deliberately rejected

- Theme-wide opacity cross-fade: the current theme switch preserves DOM, focus,
  scroll, and page state already. A shell cross-fade would add a transient
  readability dip without proving that it improves comprehension; keep this as
  a visual experiment only if a real-device review finds abrupt brightness to be
  a problem.
- Idea Tree, Activity Feed, report Markdown, metric badges, and experiment
  projection updates: users are reading durable scientific facts. Motion would
  compete with comparison and could delay trust-critical state.
- Core navigation page transitions and keyboard-driven controls: they are high
  frequency and should remain immediate.
- Motion/Framer Motion dependency: there are no drag, swipe, momentum, or
  interruptible gesture surfaces that justify a runtime animation library.
- Sound, haptics, shimmer, number tickers, and looping ambient effects: no
  product requirement or causal feedback event warrants them in this workbench.

### Verdict

The UI is already visually restrained and does not need more animation globally.
The highest-leverage defects are the Sidebar width reflow and Toast keyframes;
the remaining work is a small, targeted interaction polish pass. PR-UI-9 should
implement only the six accepted opportunities above, followed by PR-UI-10's
behavioral and accessibility regression coverage.
