### 3.0C-6 Commit 2：Bounded EnvironmentPlan Revision Loop

**目标**: 建立环境失败后的结构化修订入口和有限修订循环，确保最多 revision 2 次，不允许无限自修。

**操作**:

1. 新增 `src/autoad_researcher/environments/revision.py`
   - `EnvironmentRevisionContext`
   - `EnvironmentAttemptOutcome`
   - `EnvironmentRevisionLoopResult`
   - `build_revision_context()`
   - `run_bounded_revision_loop()`

2. Revision loop 规则
   - attempt outcome 必须匹配当前 plan_id / revision
   - failed outcome 必须包含 failure_code / failure_message
   - replacement plan 必须 parent 指向当前 plan
   - replacement revision 必须递增 1
   - replacement revision 不得超过 `permissions.max_revision_count`
   - provider 返回 `None` 时停止并返回 failed

3. Revision context
   - 从 failed build result 提取 failed_step、failure_code、failure_message
   - 从 failed validation report 提取 failed_validations
   - 生成 suggested_evidence，不提供具体修复答案

4. 更新导出与 verify gate
   - `src/autoad_researcher/environments/__init__.py`
   - `scripts/verify.sh`

5. 新增 `tests/test_environment_revision.py`
   - 一次 revision 后成功
   - 达到 revision limit 后停止
   - provider 返回 None 后停止
   - bad parent 拒绝
   - build + validation failure context
