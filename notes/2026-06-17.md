### 3.1 R6 Repository Acquisition and State Attestor

**目标**: 按 `docs/3.1开发计划文档.md` 实现受控 Git acquisition 和非 LLM Repository State Attestor；使用离线临时 Git 仓库测试 shallow / partial / local 路径，不依赖公网。

**操作**:

1. 新增 `repository_intelligence.acquisition`
   - `RepositoryAcquisitionRequest`
   - `RepositoryAcquisitionRunner`
   - `RepositoryAcquisitionResult`
   - `RepositoryAttestation`
   - `AcquisitionToolCallRecord`
   - `attest_repository()`

2. Acquisition 行为
   - `shallow_ref`：`git init` → `remote add` → `fetch --depth=1 --no-tags` → verify `FETCH_HEAD` → detached checkout → verify `HEAD`
   - `partial_exact`：`git clone --filter=blob:none --no-checkout --no-tags` → `fetch --no-tags <commit>` → detached checkout → verify `HEAD`
   - `local`：Git 目录只 attests，不 checkout / clean；非 Git 目录生成稳定 tree fingerprint
   - 禁止 credential-bearing remote URL
   - 禁止 `--filter=blob:none` 与 `--depth` 混用
   - 禁止 git submodule / lfs / config 写路径
