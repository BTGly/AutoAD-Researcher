## Entry 3 - Remove repository host allowlist from source normalization

Context:
- User asked whether Gitee/GitLab repository URLs can still clone smoothly.
- User objected that adding one more known host such as GitLab is still a brittle rule; code repository hosts are open-ended.
- Worker acquisition already has a `generic_shallow` path that can clone non-GitHub, credential-free git remotes without GitHub API metadata.

Changes:
- Removed the repository-host allowlist from source normalization.
- Explicit `.git` remotes from any HTTP(S) host are treated as repository sources without LLM.
- Non-`.git` HTTP(S) URLs remain webpages unless user/LLM intent routes them through a repository action; repository actions normalize the URL as a repo candidate without semantic correction.
- Added coverage that arbitrary `code.example.edu/.../repo.git` URLs become repository sources, while arbitrary non-`.git` URLs stay webpages until routed by repo intent.
- Worker coverage still proves non-GitHub repository sources bypass GitHub API metadata and use `generic_shallow` acquisition.
