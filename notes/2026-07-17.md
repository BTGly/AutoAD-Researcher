## PR-001B — Probe 与 ValidationContextCollector

### Goal

以真实命令和实际 Python 解释器替代 EnvironmentPlan 的预填环境声明，为后续 Worker build/validate 提供可审计的 host、repository 与 validation context。

### Operations

- 新增 host probe：OS/Python、uv/pip/conda、nvidia-smi、torch import 与 CUDA tensor compute；每条命令均为 shell=False、限时并保存 stdout/stderr。
- 新增 repository probe，复用 Repository Intelligence 的结构 profile 获取依赖文件、README、entrypoint 与 smoke 候选，并记录 git commit/dirty evidence。
- 新增 ValidationContextCollector，从实际解释器收集 runtime、package inventory、模块 import 和 torch GPU compute 结果；输出 package inventory hash。
- probe 输出应用 secret-like 文本脱敏，增加 GPU detect/compute 分离、日志脱敏、repository profile 与 observed context 测试。

### Result

- 定向环境 probe/validation/snapshot 回归：13 passed。

### Leftovers

- PR-001C 将把 probe、plan、policy、build、context、validate 接入环境准备 Worker；本 PR 不创建或执行环境 Job。

## PR-001C — Environment Prepare Job 接线

### Goal

让 `experiment_environment_prepare` Job 由 Worker 真实执行，并以 ExperimentSession 持久化环境状态、readiness 与失败结果。

### Operations

- 新增 Worker-owned preparation transaction：probe → deterministic plan → policy → build → validation context → validate。
- 将虚拟 `workspace/repos` 与 `workspace/envs` 计划路径在受控 runner 中映射到当前 run 的实际路径；子进程继续以 shell=False 执行。
- Worker 识别环境准备 Job，成功后更新 Session 为 `READY_FOR_BASELINE`，失败更新为 `ENVIRONMENT_FAILED`；未能唯一确定仓库时保留 `CREATED + readiness=blocked` 并写具体原因。
- 新增运行中 Job 的过期恢复 lease：只对超过阈值的 `running` Job 回队，记录 recovery_count，避免新 Worker 立即抢占活跃任务。
- 新增成功环境 Job、仓库歧义 blocked 与 stale-worker recovery 回归测试。

### Result

- 定向 Worker/Starter/Probe 回归：14 passed。

### Leftovers

- PR-001D 将替换 build 时的 plan-derived snapshot，生成 observed snapshot 并接入持久化 revision lineage。
