## PR-001B — Probe 与 ValidationContextCollector

### Goal

以真实命令和实际 Python 解释器替代 EnvironmentPlan 的预填环境声明，为后续 Worker build/validate 提供可审计的 host、repository 与 validation context。

### Operations

- 新增 host probe：OS/Python、uv/pip/conda、nvidia-smi、torch import 与 CUDA tensor compute；每条命令均为 shell=False、限时并保存 stdout/stderr。
- 新增 repository probe，复用 Repository Intelligence 的结构 profile 获取依赖文件、README、entrypoint 与 smoke 候选，并记录 git commit/dirty evidence。
- 新增 ValidationContextCollector，从实际解释器收集 runtime、package inventory、模块 import 和 torch GPU compute 结果；输出 package inventory hash。
- probe 输出应用 secret-like 文本脱敏，增加 GPU detect/compute 分离、日志脱敏、repository profile 与 observed context 测试。

### Result

- 定向环境 probe/validation/snapshot 回归：13 passed。

### Leftovers

- PR-001C 将把 probe、plan、policy、build、context、validate 接入环境准备 Worker；本 PR 不创建或执行环境 Job。

## PR-001C — Environment Prepare Job 接线

### Goal

让 `experiment_environment_prepare` Job 由 Worker 真实执行，并以 ExperimentSession 持久化环境状态、readiness 与失败结果。

### Operations

- 新增 Worker-owned preparation transaction：probe → deterministic plan → policy → build → validation context → validate。
- 将虚拟 `workspace/repos` 与 `workspace/envs` 计划路径在受控 runner 中映射到当前 run 的实际路径；子进程继续以 shell=False 执行。
- Worker 识别环境准备 Job，成功后更新 Session 为 `READY_FOR_BASELINE`，失败更新为 `ENVIRONMENT_FAILED`；未能唯一确定仓库时保留 `CREATED + readiness=blocked` 并写具体原因。
- 新增运行中 Job 的过期恢复 lease：只对超过阈值的 `running` Job 回队，记录 recovery_count，避免新 Worker 立即抢占活跃任务。
- 新增成功环境 Job、仓库歧义 blocked 与 stale-worker recovery 回归测试。

### Result

- 定向 Worker/Starter/Probe 回归：14 passed。

### Leftovers

- PR-001D 将替换 build 时的 plan-derived snapshot，生成 observed snapshot 并接入持久化 revision lineage。

## PR-001D — Observed Snapshot 与 Revision

### Goal

仅以真实 build、validation context 和 validation report 生成最终环境快照，并在环境失败时保留 revision context、parentage 与最多两次的后续环境 Job。

### Operations

- 移除 build 成功后由 EnvironmentPlan 声明值生成 `snapshot.json` 的行为；build result 不再把计划快照视为完成证据。
- 新增 `build_observed_environment_snapshot()`：写入实际 Python/torch/CUDA、package inventory hash、实际解释器环境路径、repository commit/fingerprint、GPU capability、validation report hash 与 project-smoke evidence。
- Worker 在 validation 通过后才写 `environment/snapshot.json` 并更新 Session 的 `environment_snapshot_ref`。
- 环境失败时写 `revision_context_rN.json`；当 Job payload 提供合法 child plan 时，原子推进 Session revision 并创建 `environment_prepare:{session_id}:r{N+1}` Job；达到 `max_revision_count=2` 后停止。
- 增加 observed-vs-forged-plan、revision 成功/上限、GPU unavailable failure code 与 baseline 不启动测试。

### Result

- 定向 snapshot/builder/fixture/Worker/revision 回归：22 passed。

### Leftovers

- 计划 01 实施完成；后续计划才涉及 baseline 与代码实验执行。

## PR-004A — Experiment Job 与 Worker dispatch

### Goal

为环境已就绪的 Session 增加可恢复的 `ExperimentAttempt` 控制面；让 baseline、exploration、confirmation 三类 PipelineJob 能经由 Worker 执行受控的单进程夹具，并保持 Attempt 运行状态与队列状态分离。

### Operations

- 新增 `ExperimentAttempt` / `ExperimentAttemptStore`：持久化 runtime status、PID/heartbeat 预留字段、超时、retry lineage、failure code、retry exhausted 和 revision；create/get、Job bind、claim/finalize 与 retry 均使用原子文件锁。
- 新增 `ExperimentAttemptService`：由 Job 类型确定 attempt purpose，要求 Session 已获授权且 baseline 仅可从 `READY_FOR_BASELINE` 创建；Attempt-first 写入后以稳定 idempotency key 补建唯一 PipelineJob。
- Worker 新增 `experiment_baseline`、`experiment_attempt` 和 `experiment_confirmatory` 分发；第一阶段继续复用现有 shell=False 短命令执行器，运行产物写入 `attempts/<attempt_id>/`，并由 Attempt 记录终态。
- 实现 retry child 的 `retry_of`、指数退避时间和独立 Job；Worker 在 `not_before` 前不 claim。仅为 attempt-scoped fixture cwd 重写 child command hash，避免重试写入父 Attempt artifact 目录。
- 参考 `/root/autodl-tmp/repos/autoresearch` 的单 GPU、单训练进程和有限 wall-time 边界（`[REFER]`，不复制代码/不引入依赖）；后续 PR-004B/04C 分别接入 ResourceLease 与 Popen 长训练。
- 添加 Attempt 重放、双 Worker claim、fixture dispatch、retry lineage/backoff/上限和 plan_only 拒绝测试。

### Result

- 定向 Attempt/runner/environment-worker 回归：22 passed。

### Leftovers

- PR-004B 将增加本地 `GpuAllocator` / `ResourceLease`；PR-004C 才将当前短命令路径替换为进程组管理的非阻塞 `Popen` 长训练。
