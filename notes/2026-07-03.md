## Entry 1 — Fix Streamlit password API key session-state loss

### Goal

Fix the L3 Streamlit UI bug where the DeepSeek API key appears injected on the Run Config page but is lost when navigating to the Preflight Executor page, causing the preflight button to remain disabled or the subprocess to report `blocked_l3_preflight_missing: env:DEEPSEEK_API_KEY`.

### Operations

- Inspected `src/autoad_researcher/ui/app.py` and confirmed the previous minimal fix removed `api_key` from `_DEFAULTS` but still bound the password widget directly to the business session-state key.
- Replaced the direct `api_key` widget binding with two separate keys:
  - `_api_key_widget` for the Streamlit password widget.
  - `_api_key_raw` for the in-memory API key consumed by the preflight executor.
- Updated the Preflight Executor page to read only `_api_key_raw`.
- Added a UI regression test that prevents reintroducing `key="api_key"` or direct `st.session_state.get("api_key")` reads.

### Result

The API key is now manually copied from the password widget into a separate in-memory session key. If Streamlit clears or desynchronizes the password widget during page navigation, the preflight executor still reads the preserved `_api_key_raw` value.
