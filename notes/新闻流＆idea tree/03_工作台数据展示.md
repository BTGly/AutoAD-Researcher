# 03. 工作台数据展示

## 1. 目标

基于后端只读投影 API，渲染实验工作台的三个区域：Session 概览、Idea Tree + 研究动态、详情面板。

页面只展示投影快照，不从 WebSocket 事件自行维护实验状态。

## 2. 设计约束

- 所有科研数据来自 `GET /api/runs/{run_id}/experiment/projection`。
- 不新增前端实验状态机。
- 底层状态值保持精确，用户界面使用确定性中文映射。
- 未知值不得猜测，必须显示原始值。
- 首版使用 React 嵌套列表，不引入 D3/Cytoscape。
- Activity Feed 展示当前投影能够可靠表达的事实，不伪造历史 mutation。
- `OutcomeCard`、`ScientificAssessment`、`AssessmentReconciliation` 分栏展示。

## 3. 页面布局

```text
┌─────────────────────────────────────────────────────────┐
│ Session 概览                                           │
│ 研究目标 · 状态 · Idea 数 · Attempt 统计 · Champion    │
├─────────────────┬──────────────────────┬────────────────┤
│ Idea Tree       │ 研究动态             │ 详情面板       │
│ 当前节点树      │ Activity Feed        │ 当前选中项     │
└─────────────────┴──────────────────────┴────────────────┘
```

无 Session 时不渲染空的三栏数据，显示空状态；`ambiguous` 时显示明确的 Session 选择提示，不自动加载任一候选。

## 4. SessionOverviewBar

新建或在 `ExperimentPage` 内拆分组件，字段来源如下：

| UI 字段 | 正确来源 |
|---|---|
| 研究目标 | `projection.input_task.user_idea`，为空则 `projection.input_task.request` |
| Baseline | `projection.input_task.baseline` |
| Dataset | `projection.input_task.dataset` |
| Primary Metrics | `projection.input_task.primary_metrics` |
| Constraints | `projection.input_task.constraints` |
| Session 状态 | `projection.session.status` |
| Environment 状态 | `projection.session.environment_status` |
| Baseline 状态 | `projection.session.baseline_status` |
| Idea 数量 | `projection.summary.idea_count` 或明确的 rooted count |
| Attempt 统计 | `projection.summary.attempt_by_status` |
| 当前 Champion | `projection.champion`；为空显示“暂未产生” |
| 预算 | `projection.summary.budget` 与 `budget_consumed` |
| 异常提醒 | failed/lost Attempt 或 readiness blockers |

`session.task_ref` 只作为开发者引用，不能显示为“研究目标”。当前 Starter 会将它固定为 `input_task.yaml`。

不要使用百分比进度条；当前数据没有可靠的总进度语义。

## 5. 中文状态映射

底层值不能改写，显示层提供映射表。映射表只接受当前 Schema 已确认的精确值：

```typescript
const ATTEMPT_STATUS_LABEL = {
  QUEUED: '等待运行',
  STARTING: '正在启动',
  RUNNING: '运行中',
  TERMINATING: '正在终止',
  COMPLETED: '已完成',
  FAILED: '运行失败',
  TIMED_OUT: '运行超时',
  CANCELLED: '已取消',
  LOST: '运行状态丢失',
} as const;
```

Idea 状态映射：

```text
DRAFT          → 草稿
REVIEWED       → 已审阅
READY          → 等待实验
RUNNING        → 实验中
SUPPORTED      → 获得证据支持
NOT_SUPPORTED  → 未获得支持
INCONCLUSIVE   → 证据不足
PRUNED         → 已停止探索
MERGED         → 已合并
```

未知值统一显示：

```text
未知状态（原始值：XXX）
```

原始值仅在开发者详情或辅助属性中保留，不从近似拼写推断中文含义。

## 6. IdeaTree

新建 `frontend/src/components/IdeaTree.tsx`，使用 `parent_id`、`children` 和 `depth` 渲染当前树。

### 6.1 展示规则

- 根节点默认展开。
- `PRUNED` 节点默认折叠，但可手动展开。
- 每个节点显示 `mechanism`；为空时使用安全的节点占位文本。
- 每个节点显示关联 Attempt 数量，不把 Attempt 复制成树节点。
- 点击节点打开详情面板。
- Champion 关联的 Idea 路径可以高亮，但只有在投影已经确定 Champion 和 Idea 关系时才高亮。
- 不从 `mutation_receipts` 反推某次历史变更。

### 6.2 节点状态视觉

| 状态 | 表现 |
|---|---|
| `DRAFT` | 空心、虚线 |
| `REVIEWED` | 默认实心 |
| `READY` | 实心 + 等待图标 |
| `RUNNING` | 运行中边框 |
| `SUPPORTED` | 绿色 |
| `NOT_SUPPORTED` | 降低透明度 |
| `INCONCLUSIVE` | 黄色虚线 |
| `PRUNED` | 灰色、默认折叠 |
| `MERGED` | 合并标记 |

这是视觉映射，不改变后端状态值。

## 7. Activity Feed

新建 `frontend/src/components/ActivityFeed.tsx`，展示投影中的 `activity`，最新在上。

卡片只使用后端已经确定的 `ActivityCard`：

- `experiment.session.created`：实验 Session 已创建；
- `experiment.idea_tree.created`：Idea 树已初始化；
- `experiment.idea_tree.mutated`：Idea Tree 已更新，树版本为当前事件 revision；
- Attempt 生命周期事件：创建、排队、运行、完成、重试；
- CognitiveCommit、Coordinator、Champion 事件：显示通用事实和可靠引用。

对 `experiment.idea_tree.mutated` 禁止显示：

- 新增了哪个 Idea；
- 哪个节点被剪枝；
- 哪个节点变成 SUPPORTED；
- 剪枝原因。

这些信息不在当前真实事件 payload 中。当前树的节点状态和 `insights` 可以在 Idea 详情中展示，但不能被包装成某一历史事件的精确结果。

### 7.1 Activity 空白与加载

- 无 Activity：显示“暂无实验动态”；
- 初始加载：显示骨架屏；
- WebSocket 事件到达：触发投影重新请求，不直接插入伪造卡片；
- 投影请求失败：保留上一份快照并显示刷新失败提示，不能清空成“无数据”。

## 8. DetailDrawer

新建 `frontend/src/components/DetailDrawer.tsx`，只展示当前选中的投影对象。

### 8.1 选中 Idea

展示：

- `mechanism`、`hypothesis`、`observable`；
- `research_axis`、`expected_cost`、`status`；
- `grounding`、`falsification`、`relationship_to_previous_ideas`；
- `evidence_refs`；
- 关联 Attempt 和 CognitiveCommit；
- 当前 `insights`，包括已记录的剪枝观察；
- 父子关系。

如果没有相关 insight，显示“暂无已记录原因”，不得从 receipt 或最终树反推原因。

### 8.2 选中 Attempt

分三个区域：

#### 执行事实

- `attempt_purpose`、`runtime_status`、`job_type`；
- command plan 摘要；
- retry 关系、`failure_code`；
- resource 和 heartbeat 信息；
- 原始 `OutcomeCard` 的执行状态、协议状态、metrics 是否解析。

#### 科学评价

- `scientific_effect`；
- `primary_delta`；
- `guardrail_deltas`；
- `evaluation_status`；
- `patch_applied`、`smoke_passed`；
- 评价 Artifact 引用。

科学评价缺失时显示：

```text
执行结果已产生，科学评价尚未物化
```

不能在页面加载时调用评价服务生成文件。

#### 权威边界

显示 `assessment_reconciliation` 对执行事实和科学比较结论的说明。

### 8.3 选中 Activity

展示：

- 标题、摘要、时间；
- 投影中已有的事件类型；
- 可靠的 Idea / Attempt / Commit 引用；
- 相关 Artifact 引用。

未知事件只在开发者详情中显示原始 JSON，不生成用户可读的猜测。

## 9. “在研究助手中讨论”

详情面板底部保留按钮：

1. 切换到 Chat；
2. 传递明确的 Idea、Attempt 或 Activity 引用；
3. 预填草稿；
4. 用户确认后才发送。

不得自动发送消息，不得把整个原始事件流塞入聊天上下文。

## 10. 开发者详情

普通模式隐藏：

- `run_id`、`session_id`；
- 原始 event type；
- Job ID、Artifact path、hash、lease ID；
- prompt version、worker 信息、stdout/stderr 路径。

展开后只读显示后端 `DeveloperRefs` 和当前选中对象的精确引用。

## 11. 验收

- 研究目标不显示 `input_task.yaml`。
- 状态显示中文且未知值保留原始值。
- Idea Tree 的父子关系和当前状态正确。
- `PRUNED` 详情只显示真实存在的 `insights`。
- Activity 不声称回放具体历史 mutation。
- Attempt 执行事实、科学评价、权威边界分开显示。
- 缺少科学评价时仍能显示执行事实。
- Champion 只显示投影已经确定的当前合同 Champion。
- “在研究助手中讨论”不会自动发送。
- 无 Session、ambiguous、加载中、请求失败都有明确界面。
