# 03. 工作台数据展示

## 1. 目标

基于后端只读投影 API，渲染实验工作台的三个核心区域：Session 概览、Idea Tree + Activity Feed、Detail Drawer。

## 2. 设计约束

- 所有数据来自 `GET /api/runs/{run_id}/experiment/projection`
- 不新增前端独立实验状态
- 状态显示使用当前 schema 的真实值，不转译、不伪造
- 首版用 React 嵌套列表渲染 Idea Tree，不引入 D3/Cytoscape

## 3. 页面布局

```
┌─────────────────────────────────────────────────────────────────┐
│  Session 概览 (SessionOverviewBar)                              │
│  任务 · 状态 · Idea 数 · Attempt 统计 · 预算 · Champion · 异常  │
├─────────────────┬──────────────────────────┬───────────────────┤
│  Idea Tree      │  Activity Feed           │  Detail Drawer    │
│  (left 30%)     │  (center 40%)            │  (right 30%)      │
│                 │                          │                   │
│  Root node      │  ┌─ 研究决策 (now) ──┐   │  (empty until     │
│  ├─ Idea 01     │  │ 新假设：...       │   │   selection)      │
│  │  (3 att.)  ──┤  └───────────────────┘   │                   │
│  ├─ Idea 02     │  ┌─ 实验执行 ────────┐   │                   │

```

## 4. SessionOverviewBar

**新建组件：** `frontend/src/components/SessionOverviewBar.tsx`

| 字段 | 来源 (`ExperimentProjection.summary`) |
|------|--------------------------------------|
| 研究目标 | `session.task_ref`（前 80 字 + "…"） |
| Session 状态 | `summary.status` |
| 环境状态 | `summary.environment_status` |
| Baseline 状态 | `summary.baseline_status` |
| Idea 数量 | `summary.idea_rooted_count` |
| Attempt 统计 | `summary.attempt_by_status`（以 badges/计数显示） |
| 当前 Champion | `champion.candidate_id` + `champion.scientific_effect`；无 → 显示「暂未产生」 |
| 预算 | `summary.budget` + `summary.budget_consumed`；无消耗 → 显示「暂无消耗」 |
| 异常提醒 | 若有 failed/lost attempt 或 readiness_blockers → 红色警告条 |

无百分比进度条。

```tsx
export function SessionOverviewBar({ projection }: { projection: ExperimentProjection }) {
  const { session, summary, champion } = projection;
  return (
    <div className="session-overview-bar">
      <div className="overview-item">
        <span className="overview-label">目标任务</span>
        <span className="overview-value">{session?.task_ref?.slice(0, 80)}…</span>
      </div>
      <div className="overview-item">
        <span className="overview-label">Session</span>
        <span className={`badge badge-${session?.status?.toLowerCase()}`}>{session?.status}</span>
      </div>
      <div className="overview-item">
        <span className="overview-label">Idea</span>
        <span className="overview-value">{summary.idea_rooted_count}</span>
      </div>
      <div className="overview-item">
        <span className="overview-label">实验</span>
        <div className="attempt-badges">
          {Object.entries(summary.attempt_by_status).map(([status, count]) => (
            <span key={status} className={`badge badge-${status.toLowerCase()}`}>
              {status}: {count}
            </span>
          ))}
        </div>
      </div>
      <div className="overview-item">
        <span className="overview-label">Champion</span>
        <span className="overview-value">
          {champion ? `${champion.candidate_id} (${champion.scientific_effect})` : '暂未产生'}
        </span>
      </div>
    </div>
  );
}
```

## 5. Idea Tree

**新建组件：** `frontend/src/components/IdeaTree.tsx`

使用 React 嵌套列表，依据 `parent_id` / `children` / `depth` 渲染。

### 节点状态视觉映射

| IdeaNodeStatus | CSS 类名 | 视觉表现 |
|----------------|----------|----------|
| DRAFT | `node-draft` | 空心圆形，虚线边框 |
| REVIEWED | `node-reviewed` | 实心，默认灰色 |
| READY | `node-ready` | 实心 + 时钟 ⏳ |
| RUNNING | `node-running` | 脉冲边框动画 |
| SUPPORTED | `node-supported` | 实心，绿色边框 |
| NOT_SUPPORTED | `node-not-supported` | 半透明，红色调 |
| INCONCLUSIVE | `node-inconclusive` | 虚线边框，黄色调 |
| PRUNED | `node-pruned` | 灰色，默认折叠 |
| MERGED | `node-merged` | 菱形 ◆ 节点 |

### 渲染逻辑

```tsx
export function IdeaTree({ tree, onSelectNode }: Props) {
  const root = tree.nodes.find(n => n.is_root);
  if (!root) return null;
  return (
    <div className="idea-tree">
      <TreeNode node={root} allNodes={tree.nodes} onSelect={onSelectNode} />
    </div>
  );
}

function TreeNode({ node, allNodes, onSelect, depth = 0 }: TreeNodeProps) {
  const children = allNodes.filter(n => node.children.includes(n.node_id));
  const isPruned = node.status === 'PRUNED';
  const [collapsed, setCollapsed] = useState(isPruned);

  return (
    <div className={`tree-node depth-${depth} ${statusClass(node.status)}`}>
      <div className="node-content" onClick={() => onSelect(node)}>
        {/* 展开/折叠图标 */}
        {children.length > 0 && (
          <button onClick={(e) => { e.stopPropagation(); setCollapsed(!collapsed); }}>
            {collapsed ? '▶' : '▼'}
          </button>
        )}
        {/* 状态图标 */}
        <span className={`node-icon node-${node.status.toLowerCase()}`} />
        {/* 标题 */}
        <span className="node-title">{node.mechanism || node.node_id}</span>
        {/* Attempt 计数 */}
        {node.attempt_summary && (
          <span className="attempt-count">
            {Object.values(node.attempt_summary).reduce((a, b) => a + b, 0)} 次实验
          </span>
        )}
      </div>
      {/* 子节点 */}
      {!collapsed && children.map(child => (
        <TreeNode key={child.node_id} node={child} allNodes={allNodes}
          onSelect={onSelect} depth={depth + 1} />
      ))}
    </div>
  );
}
```

首个 root 节点默认展开，PRUNED 节点默认折叠。

### Champion 路径高亮

当前 Champion 关联的 Idea 节点路径（根到该节点的链）添加 `.champion-path` CSS 类。

## 6. Activity Feed

**新建组件：** `frontend/src/components/ActivityFeed.tsx`

垂直时间线，最新事件在上。

### 卡片类型

```tsx
const CARD_ICONS: Record<string, string> = {
  research_decision: '🧠',
  implementation: '🔧',
  experiment: '🧪',
  result: '📊',
  failure: '❌',
  direction_change: '↗️',
  session: '📋',
};
```

### 渲染逻辑

```tsx
export function ActivityFeed({ activities, onSelectActivity }: Props) {
  return (
    <div className="activity-feed">
      <h3 className="feed-title">研究动态</h3>
      <div className="feed-timeline">
        {activities.map(activity => (
          <div key={activity.event_id} className="feed-card"
               onClick={() => onSelectActivity(activity)}>
            <span className="card-icon">{CARD_ICONS[activity.card_kind] || '📄'}</span>
            <div className="card-body">
              <div className="card-title">{activity.title}</div>
              <div className="card-summary">{activity.summary}</div>
              <div className="card-time">{formatTime(activity.created_at)}</div>
            </div>
            {/* 带 OutcomeCard 的实验结果卡片可加迷你 delta badge */}
            {activity.related_outcome?.primary_delta != null && (
              <span className={`delta-badge ${deltaSign(activity.related_outcome.primary_delta)}`}>
                {formatDelta(activity.related_outcome.primary_delta)}
              </span>
            )}
          </div>
        ))}
      </div>
    </div>
  );
}
```

### 空白与加载状态

- 无 Activity → 显示「暂无实验动态」
- 加载中 → 骨架屏（浅灰块）
- WebSocket 推送新卡片 → 插入列表顶部（由提交四驱动）

## 7. Detail Drawer

**新建组件：** `frontend/src/components/DetailDrawer.tsx`

右侧详情面板，点击 Idea / Attempt / Activity 卡片后展示对应详情。

### 7.1 选中 Idea 时

| 区域 | 内容 |
|------|------|
| 标题 | `mechanism` |
| 基本信息 | `hypothesis`、`observable`、`research_axis`、`expected_cost`、`status` |
| 科学依据 | `grounding`、`falsification`、`relationship_to_previous_ideas` |
| 证据 | `evidence_refs` 列表 |
| 关联 Attempts | `attempt_refs` 列表，每个显示 `attempt_purpose` + `runtime_status` |
| 认知提交 | `cognitive_commit_refs` 列表 |
| Insights | 按时间排列的 `IdeaInsight` |
| 父子关系 | 显示父节点标题和子节点计数 |
| 剪枝原因 | 仅 status=PRUNED 时显示，从 mutation_receipts 推断 |
| 「在研究助手中讨论」 | 按钮 → 切换到 Chat |

### 7.2 选中 Attempt 时

| 区域 | 内容 |
|------|------|
| 标题 | `attempt_id` + `attempt_purpose` |
| 运行状态 | `runtime_status`、`job_type` |
| 命令计划 | `command_plan_summary` |
| Retry | `retry_of`、`retry_count`/`max_retries`、`retry_exhausted`、`failure_code` |
| OutcomeCard | `execution_status`、`scientific_effect`、`primary_delta`、`metrics`、`evaluation_status`、`protocol_valid` |
| 资源 | `required_device_count`、`required_vram_mb`、`pid` |
| 关联 Idea | `related_idea_ids` |
| 「在研究助手中讨论」 | 按钮 → 切换到 Chat |

### 7.3 选中 Activity 卡片时

| 区域 | 内容 |
|------|------|
| 标题 | `activity.title` |
| 时间 | `activity.created_at` |
| 事件类型 | `activity.event_type` |
| 详情 | `activity.detail` |
| 关联 | Idea、Attempt、Commit 的链接（可点击） |

### 7.4 空状态

未选中任何项时，居中显示：「选择左侧的 Idea、实验或动态卡片以查看详情」

### 7.5 「在研究助手中讨论」按钮

```tsx
interface ChatContext {
  contextType: 'idea' | 'attempt' | 'activity';
  contextId: string;
  contextSummary: string;
  evidenceRefs: string[];
}

function onDiscussInChat(context: ChatContext) {
  // 1. 切换到 Chat 页面
  onSwitchToChat(context);
  // 2. Chat 页面将 context 预填为输入草稿
  // 3. 用户确认后发送
  // 4. 不得自动发送
}
```

## 8. 开发者详情

复用现有「开发者详情」折叠模式。

默认隐藏：
- `run_id`
- `session_id`
- event type 原始值
- Job ID
- artifact 路径
- prompt version
- tool calls
- stdout/stderr 路径
- hash
- lease ID
- worker 信息

展开后只读显示。

```tsx
function DeveloperDetails({ devRefs }: { devRefs: DeveloperRefs }) {
  const [open, setOpen] = useState(false);
  return (
    <details className="developer-details">
      <summary onClick={() => setOpen(!open)}>开发者详情</summary>
      {open && (
        <pre>{JSON.stringify(devRefs, null, 2)}</pre>
      )}
    </details>
  );
}
```

## 11. 验收

- Idea Tree 父子关系正确，状态图标与 schema 一致
- PRUNED 节点默认折叠，可展开
- Activity Feed 显示科研相关卡片，不显示内部事件
- 点击 Idea/Attempt/Activity 卡片，右侧显示对应详情
- OutcomeCard 缺指标时仍能展示失败事实
- 「在研究助手中讨论」切换到 Chat 页面，预填草稿
- 普通模式不显示原始路径和 ID
- 开发者详情可展开查看原始引用
- 无 Session 时显示空状态
- 加载中显示骨架屏
