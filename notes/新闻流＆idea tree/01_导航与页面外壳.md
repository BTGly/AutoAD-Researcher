# 01. 导航与页面外壳

## 1. 目标

将左侧中间「设置」页改造为「实验工作台」入口，保留所有设置能力。

## 2. 设计约束

- 不删除设置功能，继续通过顶部齿轮按钮 + `ConfigModal` 访问
- `ExperimentPage` 首版只显示空状态，不依赖后端数据
- 不修改其他页面（Chat、Report）的已有行为

### 3.5 ConfigModal 保留

ConfigModal 由顶部齿轮按钮触发，独立于页面路由。不因 SettingsPage 移出主路由而丢失。

```tsx
// App.tsx 顶部
<button onClick={() => setShowConfig(true)}>⚙</button>
{showConfig && <ConfigModal onClose={() => setShowConfig(false)} />}
```

### 3.6 Session 选择规则

`ExperimentSessionStore` 按 `session_id` 读取，不允许按文件时间猜测。

| 情况 | 前端行为 |
|------|----------|
| 一个 run 没有 Session | 显示空状态：实验尚未启动 |
| 一个 run 只有一个 Session | 自动选中 |
| 一个 run 有多个 Session | 由用户在下拉列表中选择 |
| 不允许 | 根据文件名、修改时间或目录顺序暗中选择 |

## 5. 验收

- 左侧导航显示「实验工作台」，图标 🔬
- 点击后打开 ExperimentPage，无 Session 时显示空状态
- 点击「前往研究助手」切换到 Chat 页面
- 顶部齿轮按钮仍可打开 ConfigModal
- 设置功能未丢失
- 现有 Chat、Report 页面不受影响
