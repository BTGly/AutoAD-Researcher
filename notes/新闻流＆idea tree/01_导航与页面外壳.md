# 01. 导航与页面外壳

## 1. 目标

将左侧“设置”入口改成“实验工作台”，同时保留现有实验配置能力和通用 API 配置能力。

## 2. 当前配置事实

当前代码中存在两套不同职责的组件，不能把它们描述成同一套表单：

| 组件 | 当前职责 | 首版处理 |
|---|---|---|
| `ConfigModal` | 通用 API Key、Base URL、Model | 继续由顶部齿轮打开 |
| `SettingsPage` | 实验 Provider、实验 Model、实验 API Key、Reasoning Effort、Max Cycles、Max Turns、Executor Timeout、Literature Search、Idea Novelty Search | 保留表单，嵌入实验工作台的“实验配置”入口 |

`SettingsPage` 目前通过 `useConfig().saveExperimentConfig` 保存实验配置；不能因为移出主路由而丢失这些字段。
`ConfigModal` 并没有这些实验字段，因此不能写成“ConfigModal 复用 SettingsPage 的全部内容”。

## 3. 设计约束

- 不删除 `SettingsPage` 的实验配置功能。
- 不重写第二套实验配置表单。
- 顶部齿轮仍然只打开现有 `ConfigModal`。
- 实验工作台中的“实验配置”按钮复用现有 `SettingsPage` 表单，可采用弹窗、抽屉或嵌入区域；具体容器必须先核对当前组件 Props。
- `ExperimentPage` 首版先支持空状态和投影加载入口，不在页面内复制实验核心状态。
- Chat、Report 的现有行为不改变。

## 4. 改动清单

### 4.1 `frontend/src/lib/types.ts`

将现有 `PageId` 的 `settings` 页面替换为 `experiment`。这是新的前端路由类型，不是后端 Schema 字段。

编码前必须读取当前文件，精确保留其他 `PageId` 成员和 `WSMessage` 字段。

### 4.2 `frontend/src/components/LeftSidebar.tsx`

仅将设置入口的展示文案和 ID 改成实验工作台；继续沿用当前侧栏布局和状态切换方式。

建议显示：

```text
🔬 实验工作台
```

不要在这里增加 Session 发现、实验状态机或 API 请求。

### 4.3 `frontend/src/App.tsx`

- 将主路由从 SettingsPage 切换到 ExperimentPage。
- 顶部齿轮仍使用当前 `ConfigModal config={config}` 路径。
- 通过一个实验配置入口打开现有 `SettingsPage`，继续传入当前 `experiment`、`defaultApiKey`、`onSave` 和返回回调。
- 不要把 `SettingsPage` 的实验字段合并进 `ConfigModal`，除非后续另有明确产品决策。

### 4.4 `frontend/src/components/ExperimentPage.tsx`

新建页面外壳，首版只负责布局、空状态、投影加载和配置入口：

```text
顶部：Session 概览
左侧：Idea Tree
中间：研究动态
右侧：详情面板
```

空状态：

```text
实验尚未启动。请先在“研究助手”中确认实验任务。
```

无 `runId` 时显示：

```text
请先创建一个研究任务。
```

页面收到后端投影的 `no_session`、`selected`、`ambiguous` 状态后分别处理：

- `no_session`：显示空状态；
- `selected`：加载选中的投影；
- `ambiguous`：显示“发现多个实验 Session，请明确选择”，但首版不依据文件顺序自动选择。

多 Session 选择控件是否实现，必须等后端投影 Schema 定稿后再决定；不能先在前端猜测接口。

## 5. 实验配置入口

页面顶部或概览区域增加“实验配置”按钮。按钮打开现有 `SettingsPage` 表单的嵌入容器：

1. 读取 `config.experiment ?? DEFAULT_EXPERIMENT`；
2. 继续传入现有 `saveExperimentConfig`；
3. 保存后关闭容器并保留当前工作台选择状态；
4. 不自动启动实验；
5. 不自动发送聊天消息。

## 6. 验收

- 左侧显示“实验工作台”，点击后进入 ExperimentPage。
- 顶部齿轮仍能打开通用 `ConfigModal`。
- “实验配置”仍能修改 `SettingsPage` 当前的全部实验配置字段。
- SettingsPage 的保存仍写入现有 `autoad_config` 结构。
- 无 `runId`、无 Session、多 Session 都有明确状态。
- 不根据文件名、修改时间或目录顺序偷偷选择 Session。
- Chat、Report 页面行为不回归。
