### Step 2.8: Minimal PipelineController

**目标**: 建立最小 pipeline 编排器，串联 experiment_planning + patch_planning。

**操作**:

1. 创建 `core/pipeline.py`
   - `PipelineResult(run_id, status, stages)` — 一次 pipeline 的结构化结果
   - `PipelineStatus = Literal["success", "failed"]`
   - `PipelineController(harness, runs_root)` — 依赖 AgentHarness 接口，不绑定具体实现

2. `PipelineController.run_planning_pipeline(run_id)`
   - 校验 run_id + 创建 run_dir
   - 写 `run_created` 事件（payload 含 pipeline name）
   - 依次调用 harness.run_experiment_planning() → harness.run_patch_planning()
   - 收集 StageResult 列表，返回 PipelineResult

3. 更新 `core/__init__.py` + `verify.sh` core import

4. 测试
   - `tests/test_pipeline_controller.py` — 3 tests：成功路径 / 事件顺序 / 非法 run_id

**结果**:
- 57 passed（+3）

**关键设计边界**:
- Controller 管 run 级生命周期（run_created），Harness 管 stage 级，ArtifactStore 管 artifact 级
- 不 catch exception — 失败直接抛出，留给 Step 2.9
- 不接 DeepAgents 真实测试 — Controller 只依赖 AgentHarness 抽象接口
- PipelineResult extra="forbid"

**事件顺序**:
```
run_created → stage_started → artifact_written → stage_completed
            → stage_started → artifact_written → stage_completed
```
