## Entry 6 — Derive readable paper context and reject garbled parse artifacts

### Goal

继续修复 Research Chat “再读论文”场景：不能只因 `paper_summary.json`、`sections.json` 或 `blocks.jsonl` 文件存在就告诉 LLM 可读；必须把 MinerU `blocks.jsonl` 派生为稳定的 `paper.md`，并把真正可读内容注入 `paper_context`。对于真实 run 中 `paper_summary.title`、`sections.title`、`blocks.text` 都是 PDF 二进制/短 token 乱码的情况，应明确 `paper_context.can_answer_from_paper=false`，避免模型继续说已经读到论文内容或被旧 transcript 诱导反复要求 arXiv HTML。

### Operations

- 新增 `src/autoad_researcher/paper_intelligence/markdown.py`：
  - 从 `blocks.jsonl` 生成派生 artifact `paper.md`。
  - 跳过乱码 block，不改原始 `blocks.jsonl`。
  - 用非 ASCII/控制字符/短 token 分布/正文词密度检测 PDF 二进制碎片。
- 在 `PaperIntelligenceOrchestrator` parse 后调用 `blocks_jsonl_to_paper_markdown()`，生成 `paper/parse/attempts/<pa_id>/paper.md`；active parse snapshot 同步时会把它复制到 `paper/parse/paper.md`。
- 在 `ResearchContextSnapshot` / ResponseContext facts 中新增：
  - `unreadable_artifacts`
  - `paper_context`
- 重写可读判断：
  - `paper_summary.json` 只从真实 claim 内容提取，不再被 `claim_id`、`status`、`evidence_ids` 等短字段误导。
  - `paper.md` 只有正文密度足够时才进入 readable。
  - `sections.json` 只有正文存在，或标题是常见论文章节名时才作为可读章节；默认 `Section`、`Adobe d`、短 token 标题不再算可读内容。
  - `blocks.jsonl` 有乱码时进入 `unreadable_artifacts`，但不作为 `readable_artifacts`。
- 在自然回复 prompt 中改为优先 `ResponseContext.facts.paper_context`：
  - `can_answer_from_paper=true` 时基于 `paper_context`、`paper.md`、`paper_summary.json`、`sections.json` 回答论文内容。
  - `can_answer_from_paper=false` 时不得假装读过论文，应说明当前 artifact 仍无可读论文文本。
  - 当前 ResponseContext 优先级高于旧 transcript。
- 对 `_natural_reply_for_decision()` 增加旧历史过滤：当当前 `paper_context` 可回答时，过滤旧 assistant 历史里的“无法读取 / arXiv HTML / 正文乱码”等过期回复，避免 LLM 复读旧错误结论。
- 更新 prompt registry 与 UI base prompt，移除 “看到 blocks.jsonl 就不要说无法读取” 这种过宽规则，改为 `paper_context` 驱动。
- 补充测试：
  - `paper.md` 生成跳过乱码 block。
  - 只有乱码 summary/sections/blocks 时 `paper_context.can_answer_from_paper=false`。
  - 可读 blocks 生成 `paper_markdown_excerpt` 并注入 `paper_context`。
  - 自然回复 prompt 使用 `paper_context/paper.md`。
  - 当前 `paper_context` 可读时过滤旧的“无法读取 PDF / arXiv HTML”历史。
