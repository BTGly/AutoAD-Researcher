## Entry 2 - Add typed source reparse candidate and V2 permission dispatch

Goal:
- Implement the first bounded typed source action from the V2 decision design:
  request a new immutable parse attempt for an already-registered PDF source.
- Route source actions through the existing PermissionEngine instead of using
  an LLM response as authorization.

Operations:
- Extended `SourceInstruction` with `request_source_reparse`; source removal
  remains a separate destructive action.
- Added a narrow V2 source-action permission adapter. It allows only
  `paper_parse_mineru` for a registered local PDF and returns `ask` for source
  removal. Code, experiment, and system actions remain outside the V2 dialogue
  dispatcher.
- Updated `DialogueGate` to validate an exact source id and reparseable PDF,
  append a permission audit record, and strip denied or unavailable actions.
- Added reparse dispatch after a validated Reply summary. It records a
  `source.reparse_queued` event and suppresses an equivalent queued/running
  parse job for the same source.
- Added Gate and orchestration regressions for allowed reparse, unavailable
  source input, preserved old parse attempt metadata, permission audit, and
  duplicate-job suppression.
- Updated the Reply profile so an authorized source reparse is described as a
  material operation rather than as code or experiment execution.

Result:
- A user-requested reparse can now traverse Decision -> Gate ->
  PermissionEngine -> job ledger without loosening execution boundaries.
- Focused regression suite: 55 passed.

Leftovers:
- Add the full orthogonal decision schema and persisted state projection.
- Convert the 57-case runner to declarative preconditions and multi-axis
  assertions after the state projection is available.

## Entry 3 - Persist bounded V2 dialogue state projection

Goal:
- Replace transcript-only continuity with a bounded projection assembled from
  persisted gated decisions, source jobs, TaskBridge state, and source parse
  state.
- Ensure an invalid decision or an unpersisted Reply never becomes a recorded
  dialogue transition.

Operations:
- Added the V2 transition ledger under `assistant/v2_dialogue_transitions.jsonl`.
  Records include the gated mode/policy, candidate source/task action,
  permission outcome, execution gate, gate notes, and summary hash; no raw
  user text, provider output, or host path is written.
- Added a read-only `dialogue_state` projection to the existing one-per-turn
  context build. It supplies the last persisted decision, bounded pending
  source actions, TaskBridge status, and compact source/parse state to both
  Decision and Reply calls.
- Appended a transition only after the validated Reply summary is saved. A
  schema-invalid Decision continues to preserve the previous summary and now
  also leaves no transition record.
- Added regressions for transition persistence, pending reparse action
  visibility, TaskBridge status precedence, compact source parse state, and
  failed-decision non-persistence.

Result:
- Multi-turn V2 decisions now have a persisted control-plane continuity input
  without making the transcript the authority for source, task, or action
  state.
- Focused regression suite: 75 passed, 2 skipped.

Leftovers:
- Migrate the Decision schema from overloaded mode/policy fields to the
  orthogonal contract in the design document.
- Rebuild the 57-case fixture and scoring layer on the completed decision,
  permission, and state projections.
