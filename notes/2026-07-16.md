#### 2. Champion 晋升系统重构

经三次逐轮决策，确定了 Champion promotion 的最终设计：

- 命令/事务/事件三层分离
- PromotionTransaction 两阶段提交（PREPARED → GIT_APPLIED → COMMITTED）
- 崩溃恢复规则覆盖 PREPARED/GIT_APPLIED/COMMITTED 三种路径
- `git merge --no-ff` + `git revert -m 1` 策略
- HITL 默认 + auto-approve 显式开启 + 14 项硬门禁条件
- V1 仅暴露 `promote_and_merge_candidate`，`PROMOTE_ONLY/MERGE_ONLY` 仅 schema 预留

| 议题 | 决策 |
|------|------|
