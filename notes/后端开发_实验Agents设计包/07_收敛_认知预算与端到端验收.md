### 3.2 本轮实现：Step Signature（参考 MiMo 算法，自行移植）

> 参考 MiMo `stableStringify()` 和 `stepSignature()` 的算法语义（`[PORT-PENDING-LICENSE]`——mimo-code 仓库许可证确认后方可落地）。

```python
REPEATED_STEP_THRESHOLD = 3  # 连续 3 次相同 → 触发

def stable_stringify(tool_name: str, args: dict) -> str:
    return json.dumps({"tool": tool_name, "args": args}, sort_keys=True)

def step_signature(tool_calls: list[ToolCall]) -> str:
    segments = []
    for call in tool_calls:
        segments.append(f"tool:{call.tool_name}:{stable_stringify(call.args)}")
    return "\n".join(segments) if segments else ""
```

**适用：** Coordinator 用同一 tool+参数连续调用 3 次；Executor 用同一 SEARCH/REPLACE block 连续提交 3 次。

### 3.3 本轮额外实现：AutoAD 特定检测

| 检测 | 触发条件 |
|------|----------|
| repeated same patch | patch.diff SHA256 连续 2 次未变 |
| repeated same failing command | subprocess exit_code 相同且 stderr 内容相同 |

不实现 monologue detection、alternating pattern detection 和 context window error detection。

### 3.4 延后：OpenHands 5-Mode StuckDetector（`[ADAPT-LATER]`）

OpenHands `stuck_detector.py`（320 行）的算法有参考价值，但依赖 OpenHands SDK 的事件类型系统（`ActionEvent`、`ObservationEvent`、`AgentErrorEvent` 等），不能直接复制。

完整 5-mode 检测延后到主闭环完成后适配。

| Mode | 检测什么 | 适配前提 |
|------|----------|---------|
| 1. Repeating Action-Observation | 同一 tool + 同一返回内容重复 N 次 | 需定义 AutoAD 通用 Event 类型 |
| 2. Repeating Action-Error | 同一 tool + 错误重复 N 次 | 同上 |
| 3. Monologue | 连续 N 条消息无 tool call | 需 Agent 框架支持消息流事件 |
| 4. Alternating ABAB | A→B→A→B 交替模式 | 需完整 Agent turn 记录 |
| 5. Context Window Error | 重复 context overflow | DeepAgents 框架抽象后 |

## 7. 全链路测试基准

建立小型 `AD-AgentBench` fixture，不追求模型 SOTA，测试闭环能力。

### Case 1：明确有效的小参数优化

- baseline 可运行；
- 一个参数变化提升超过 noise；
- 预期：candidate→confirm→champion。

### Case 2：实现未生效

- patch 成功；
- 实际路径未触发；
- 预期：smoke_passed=False → bounded repair，不 prune idea。

### Case 3：实现有效但指标下降

- patch_applied + smoke_passed + metrics_parsed + protocol_intact；
- 预期：NOT_SUPPORTED；
- 写 failure-WHY；
- 派生或 pivot。

### Case 4：结果在噪声内

- 预期：补 seed 或 NO_EFFECT；
- 不提升 champion。

### Case 5：GPU OOM

- PostRunFailureClassifier 分类；
- 无 LLM 实时监控；
- bounded repair/retry。

### Case 6：训练 hang

- stale heartbeat；
- TERM/KILL；
- Job 有明确状态；
- ResourceLease 释放。

### Case 7：Coordinator 重启

- 删除 checkpoint；
- 从 Session/Tree/Commit 恢复；
- 不重复已完成 Job。

### Case 8：便宜实验批次

- 一次生成 3 variants；
- 3 个 Result Integration；
- 批次结束后一次 Compact Cycle。

### Case 9：停滞

- 重复 idea / low KEEP；
- StrategyDiagnostic；
- overlay；
- 新方向出现；
- prompt change 有审计。

### Case 10：评估作弊

- 修改 eval/split；
- protected guard 拒绝；
- 结果不进入 DecisionEngine。

## 10. 端到端验收标准

系统只有满足以下条件才算“实验 Agents 第一版完成”：

1. 能从已确认任务创建 Session；
2. 能自动准备并验证环境；
3. 能运行 baseline；
4. Coordinator 能基于 baseline 连续 ideate；
5. Executor 在 worktree 修改；
6. GPU Job 可恢复、取消和超时；
7. 训练正常时不调用监控 LLM；
8. 实现生效与评估可比较有证据；
9. noise floor 参与决策；
10. 每个研究决策有 CognitiveCommit；
11. checkpoint 丢失后可恢复；
12. 能完成至少两轮结果驱动的假设更新；
13. 能区分 repair、new child idea 和 inconclusive；
14. 能检测重复、停滞和预算；
15. 所有关键操作有 artifact/event/hash；
16. protected artifact 被改时结果自动失效；
17. cheap batch 下认知成本可控；
18. 不依赖前端或最终报告 Agent 才能运行。
